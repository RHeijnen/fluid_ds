`<fluid-loading-overlay>` wraps a region of your UI and, while `active`, covers
it with a scrim plus a [`<fluid-spinner>`](/components/spinner/) and an optional
`label`. It traps pointer events so the gated controls cannot be operated and
marks the host `aria-busy`, announcing the busy state to assistive technology
via a polite `role="status"` region.

<Demo>
  <fluid-loading-overlay active label="Loading account…">

      <h3 style="margin: 0 0 8px;">Account summary</h3>
      <p style="margin: 0; color: var(--fluid-text-secondary);">Your balance and recent activity load here.</p>
    </div>
  </fluid-loading-overlay>
</Demo>

### Toggling around an async task

Set `active` before the work starts and clear it when the promise settles.

          setBusy(false);
        }

## When to use

- Gating a panel, card, or form section while it refreshes in place.
- Blocking interaction with a region during an async submit.
- Showing a busy state that does not need to interrupt the whole page.

Reach for [`<fluid-dialog>`](/components/dialog/) when the operation must block
the entire page, and for [`<fluid-skeleton>`](/components/skeleton/) when you are
showing the shape of content that is loading for the first time rather than
gating existing content.

## Theming

Every styled property reads a **component-scoped `--fluid-loading-overlay-*`
token that falls back to a main semantic var** (the override ladder). The scrim
defaults to a translucent surface so the gated content stays faintly visible
underneath. The **complete list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] fluid-loading-overlay {
  --fluid-loading-overlay-scrim: color-mix(in srgb, var(--fluid-surface-base) 80%, transparent);
  --fluid-loading-overlay-blur: 4px;
  --fluid-loading-overlay-fg: var(--fluid-text-primary);
}
```

### Beyond tokens: `::part()`

The wrapper is [`part="base"`](#api) with `part="overlay"` for the scrim layer,
`part="spinner"` for the loader, and `part="label"` for the text. Reach them
with [`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for
CSS not covered by a token.

## Accessibility

- The overlay layer is `role="status"`, a polite live region, so the busy
  state is announced without interrupting the user. The host is also marked
  `aria-busy="true"` while `active`.
- The status region carries an accessible name from `label`, falling back to
  "Loading" when no label is set, so the announcement is never silent.
- The spinner is `aria-hidden` because the status region already conveys the
  busy state, this avoids a doubled announcement.
- While `active`, the overlay sits above the gated content and traps pointer
  events so the controls underneath cannot be operated.
- The fade-in honors `prefers-reduced-motion` and the global `--fluid-motion`
  scalar.

### AA vs AAA

The overlay renders no interactive targets of its own, so target-size criteria
do not apply to it. Under `data-fluid-conformance="aaa"`, the controls you place
inside the gated content inherit the enhanced target-size and focus tokens as
usual; the overlay does not need per-component branching. Ensure the scrim's
`label` text keeps at least 4.5:1 contrast against the scrim color if you
retune `--fluid-loading-overlay-scrim` and `--fluid-loading-overlay-fg`.
