`<fluid-card>` groups related content into a visual block, header, body,
optional footer. It's the workhorse surface for dashboards, settings panels,
and anything that benefits from a clear bounding box. Two visual variants
(`default` and `outline`) plus a `compact` size let it slot into dense lists
without overpowering the layout.

<Demo>
  <fluid-card style="max-width: 22rem;">
    <h3 slot="header" style="margin: 0;">Hello, world</h3>
    <p style="margin: 0;">
      Cards keep your content visually grouped. The header and footer slots are
      optional, leave them out for a body-only block.
    </p>
  </fluid-card>
</Demo>

### Outline variant

The `outline` variant drops the shadow in favour of a one-pixel border, use
it when cards are nested or when an extra shadow would feel heavy.

## When to use

- Grouping multiple related controls or pieces of content.
- Modal-style or dialog-style blocks that aren't truly modal (use
  [`<fluid-dialog>`](/components/dialog/) for actual modals).
- List items where each row needs its own header/footer rhythm.

If you only need a thin separator without grouping, reach for
[`<fluid-divider>`](/components/divider/).

## Theming

Every styled property reads a **component-scoped `--fluid-card-*` token that
falls back to a main semantic var** (the override ladder), so a brand can
retune the card without touching the surface track everywhere else. The
**complete list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-card-bg: var(--fluid-surface-base);
  --fluid-card-border: var(--fluid-border-default);
  --fluid-card-padding: 1.5rem;
}
```

### Beyond tokens: `::part()`

The card exposes [`part="base"`](#api) plus `part="header"`, `part="body"`, and
`part="footer"`. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token, e.g. a sticky header inside a scrolling card.

## Accessibility

- The card is a passive grouping surface: no implicit ARIA role. If your
  card is interactive (e.g. an entire card is clickable), wrap it in a
  `<button>` or anchor and set a meaningful `aria-label`.
- Use a proper heading element (`<h2>`, `<h3>`, etc.) inside the header
  slot so the page's heading outline stays coherent.
