`<fluid-speed-dial>` is a floating action button (FAB) that fans out a small set
of actions when activated. It implements the WAI-ARIA Menu Button pattern: the
trigger carries `aria-haspopup="menu"` and `aria-expanded`, and the actions form
a `role="menu"` of `role="menuitem"` buttons. The dial closes on Esc (returning
focus to the trigger), on outside click, and after an action fires
**`fluid-action`**.

### Reacting to an action

Each action is a slotted button. The dial fires **`fluid-action`** when one is
activated (by click or keyboard); `event.detail.action` is the activated
element. The dial closes automatically afterward.

## When to use

- A small, frequently used set of related actions anchored to a corner of the
  screen (the classic floating action button).
- Compact toolbars where the actions should stay out of the way until summoned.

Reach for a plain [`<fluid-button>`](/components/button/) when there is a single
primary action, and for [`<fluid-select>`](/components/select/) when the user is
choosing a value rather than triggering an action.

## Theming

Every styled property reads a **component-scoped `--fluid-speed-dial-*` token
that falls back to a main semantic var** (the override ladder). The **complete
list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] fluid-speed-dial {
  --fluid-speed-dial-bg: var(--fluid-accent-base);
  --fluid-speed-dial-fg: var(--fluid-accent-text);
  --fluid-speed-dial-size: 4rem;
  --fluid-speed-dial-gap: var(--fluid-space-4);
}
```

### Beyond tokens: `::part()`

The wrapper is [`part="base"`](#api), with `part="trigger"` for the floating
button and `part="menu"` for the actions container. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for any CSS
not covered by a token.

## Accessibility

- Implements the
  [Menu Button pattern](https://www.w3.org/WAI/ARIA/apg/patterns/menu-button/):
  the trigger is a real `<button>` with `aria-haspopup="menu"` and
  `aria-expanded`, and the actions are a `role="menu"` of `role="menuitem"`
  children.
- The trigger has no visible text, so it carries `aria-label` (set with the
  `label` prop). Give it a descriptive value like "Quick actions".
- Keyboard: Enter / Space / the placement-aligned arrow opens the menu and moves
  focus into it; arrow keys move between actions (roving tabindex); Home / End
  jump to the first / last action; Esc closes the dial and returns focus to the
  trigger; Tab closes it.
- The trigger and every action meet the 24×24 CSS px minimum target size and
  scale to 44×44 under `data-fluid-conformance="aaa"`.
- The fan-out animation honors `prefers-reduced-motion` and the `--fluid-motion`
  scalar.

### AA vs AAA

Under the default AA conformance, the trigger uses its design diameter and a 2px
focus ring. When an ancestor sets `data-fluid-conformance="aaa"`, the trigger's
minimum target lifts to 44×44 px (via `--fluid-target-min`) and the focus ring
widens (via `--fluid-focus-ring-width`), with no per-component changes.
