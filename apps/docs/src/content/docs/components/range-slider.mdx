`<fluid-range-slider>` lets a user pick a **range** (a low and a high value)
with two draggable thumbs on one rail. The selected span is filled between the
thumbs, the thumbs can never cross, and it submits its value as
`"valueMin,valueMax"` like any native form control. It is the two-handled
sibling of [`<fluid-slider>`](/components/slider/).

## Reacting to changes

Both thumbs report through two events:

- **`fluid-input`** fires on every change during a drag or key press.
- **`fluid-change`** fires once the change is committed (pointer release or key
  press).

Both carry `detail: { min, max }` with the current thumb values.

## When to use

- Price, date, or numeric **range** filters (between a low and a high bound).
- Any "from X to Y" selection where both ends are adjustable.

Reach for [`<fluid-slider>`](/components/slider/) when you only need a single
value, not a span.

## Theming

Every styled property reads a **component-scoped `--fluid-range-slider-*` token
that falls back to a main semantic var** (the override ladder): the rail, the
selected-range fill, and the thumbs each have their own token. The **complete
list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-range-slider-track-color: var(--fluid-surface-muted);
  --fluid-range-slider-fill-color: var(--fluid-accent-base);
  --fluid-range-slider-thumb-color: var(--fluid-accent-base);
}
```

### Beyond tokens: `::part()`

The container is [`part="base"`](#api) with `track`, `fill`, `thumb` (both
thumbs), and the individual `thumb-min` / `thumb-max` parts. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

Built to the WAI-ARIA APG
[Multi-Thumb Slider](https://www.w3.org/WAI/ARIA/apg/patterns/slider-multithumb/)
pattern.

- Each thumb is its own `role="slider"` with `aria-valuemin`, `aria-valuemax`,
  and `aria-valuenow`. The names are **"Minimum"** and **"Maximum"** so screen
  readers can tell the two apart.
- The thumbs cannot cross: the minimum thumb's `aria-valuemax` is the current
  maximum value and the maximum thumb's `aria-valuemin` is the current minimum,
  so assistive tech always announces the true allowed range.
- **Keyboard**: Tab moves between thumbs. On a focused thumb, Arrow Right / Up
  increase and Arrow Left / Down decrease by `step`; Page Up / Page Down move by
  a larger step; Home / End jump to that thumb's allowed bound.
- Set `valueFormatter` to surface a human-readable `aria-valuetext` (for
  example `$200`) on both thumbs.

### AA vs AAA

The thumb pointer-target row reads `--fluid-target-min`, so it grows from the
AA 24×24 floor to a 44×44 hit area when an ancestor opts into
`data-fluid-conformance="aaa"`, and the focus ring widens via
`--fluid-focus-ring-width`. No per-instance change is needed.
