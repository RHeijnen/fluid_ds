`<fluid-select>` is the dense single-select picker, a button trigger that opens
a listbox of `<fluid-option>` children. It's form-associated and implements the
WAI-ARIA combobox + listbox pattern, with arrow-key navigation, type-to-jump,
and roving-tabindex focus inside the listbox.

<Demo>
  <fluid-select value="us" aria-label="Country" style="max-width: 18rem;">
    <fluid-option value="us">United States</fluid-option>
    <fluid-option value="ca">Canada</fluid-option>
    <fluid-option value="mx">Mexico</fluid-option>
    <fluid-option value="uk">United Kingdom</fluid-option>
    <fluid-option value="de">Germany</fluid-option>
  </fluid-select>
</Demo>

## Listening for changes

The select fires **`fluid-change`** with `{ value }` in `detail` when the
selection changes.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-select id="country" aria-label="Country">…</fluid-select>

    <script type="module">
      document.getElementById("country").addEventListener("fluid-change", (e) => {
        console.log("selected:", e.detail.value);
      });
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="CountryPicker.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/select";

    export function CountryPicker() {
      const ref = useRef<HTMLElement>(null);
      useEffect(() => {
        const el = ref.current!;
        const onChange = (e: Event) =>
          console.log((e as CustomEvent).detail.value);
        el.addEventListener("fluid-change", onChange);
        return () => el.removeEventListener("fluid-change", onChange);
      }, []);
      return (
        <fluid-select ref={ref} aria-label="Country">
          <fluid-option value="us">United States</fluid-option>
          <fluid-option value="ca">Canada</fluid-option>
        </fluid-select>
      );
    }
    ```
  </TabItem>
  <TabItem label="Vue">
    ```vue title="CountryPicker.vue"
    <script setup lang="ts">
    import "@fluid-ds/components/define/select";
    const onChange = (e: CustomEvent) => console.log(e.detail.value);
    </script>

    <template>
      <fluid-select aria-label="Country" @fluid-change="onChange">
        <fluid-option value="us">United States</fluid-option>
        <fluid-option value="ca">Canada</fluid-option>
      </fluid-select>
    </template>
    ```
  </TabItem>
  <TabItem label="Angular">

## Inside a form

The select is form-associated: it submits `name=<value>`, and `required` blocks
submission until a choice is made.

<Demo>
  <form
    style="display:flex; flex-direction:column; gap:0.75rem; align-items:flex-start; max-width:18rem;"
    onsubmit="event.preventDefault(); this.querySelector('output').textContent = 'role=' + (new FormData(this).get('role') ?? '(none)');"
  >
    <fluid-select name="role" required placeholder="Pick a role…" style="width:100%;">
      <fluid-option value="admin">Admin</fluid-option>
      <fluid-option value="editor">Editor</fluid-option>
      <fluid-option value="viewer">Viewer</fluid-option>
    </fluid-select>

## Theming

Every styled property of the select reads a **component-scoped `--fluid-select-*`
token that falls back to a main semantic var** (the override ladder), most fall
through to the shared `--fluid-field-*` field tokens so a select matches your
inputs by default. The **complete list** is in the [API tables below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-select-bg: var(--fluid-surface-base);
  --fluid-select-border: var(--fluid-border-default);
  --fluid-select-border-focus: var(--fluid-accent-base);
  --fluid-option-selected-bg: var(--fluid-accent-base);
  --fluid-option-selected-fg: var(--fluid-accent-text);
}
```

<Aside type="tip">
  Select a `fluid-select` in the [Theme builder](/playground/) to tweak the
  trigger, listbox, and option tokens visually.
</Aside>

### Beyond tokens: `::part()`

The trigger is [`part="trigger"`](#api) and the listbox is `part="listbox"`.
Reach them with [`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part)
for CSS not covered by a token.

## When to use

- **One-of-many** selection where the option set is finite and known in advance.

### When *not* to use

- **Free-form text plus suggestions**: use
  [`<fluid-typeahead>`](/components/typeahead/).
- **2–5 always-visible options**: use
  [`<fluid-segmented-control>`](/components/segmented-control/) or
  [`<fluid-radio>`](/components/radio/).
- **A menu of actions** (not a value): use
  [`<fluid-dropdown>`](/components/dropdown/).

## Accessibility

The trigger is `role="combobox"` and the popup is `role="listbox"`, following
the WAI-ARIA APG. The notes below are select-specific; the cross-component
contract is in the [Accessibility guide](/guides/accessibility/).

### Keyboard

| Key | Action |
| --- | --- |
| <kbd>Space</kbd> / <kbd>Enter</kbd> / <kbd>↓</kbd> | Open the listbox; focus the selected (or first) option. |
| <kbd>↑</kbd> / <kbd>↓</kbd> | Move the active option (skips disabled). |
| <kbd>Home</kbd> / <kbd>End</kbd> | Jump to the first / last option. |
| _type a character_ | Jump to the next option starting with it. |
| <kbd>Enter</kbd> / <kbd>Space</kbd> | Select the active option and close. |
| <kbd>Escape</kbd> | Close without changing the value. |

Closing returns focus to the trigger.

### Names & roles

The trigger carries `aria-haspopup="listbox"` and `aria-expanded` reflecting the
open state. Each option is `role="option"` with `aria-selected` reflecting the
current value. The select needs an accessible name, a visible label wired with
`aria-labelledby`, or an `aria-label`
([SC 4.1.2](https://www.w3.org/WAI/WCAG22/Understanding/name-role-value)).

### State semantics

`required` participates in form validation; the active option is tracked with
`aria-activedescendant` so screen readers announce navigation without moving DOM
focus.

### Motion & target size

- Open/close and option transitions honor `prefers-reduced-motion`.
- The trigger reads `--fluid-target-min` as a height floor: ≥ **24×24** (AA) /
  **44×44** (AAA), so even the `sm` size stays a comfortable hit target.

### AA vs AAA

Ships **WCAG 2.2 AA**; flip the **Conformance** toggle to **AAA**:

| Token | AA | AAA | Criterion |
| --- | --- | --- | --- |
| `--fluid-target-min` | 24px | **44px** | 2.5.8 → [2.5.5 Target Size (Enhanced)](https://www.w3.org/WAI/WCAG22/Understanding/target-size-enhanced) |
| `--fluid-focus-ring-width` | 2px | **3px** | 2.4.7 → [2.4.13 Focus Appearance](https://www.w3.org/WAI/WCAG22/Understanding/focus-appearance) |
