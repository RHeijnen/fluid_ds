`<fluid-number-input>` is the numeric variant of
[`<fluid-input>`](/components/input/), the same field shell plus dedicated
increment/decrement buttons. It enforces `min` / `max` / `step`, formats the
value with `tabular-nums`, is **form-associated** (submits + validates like a
native `<input type="number">`), and meets the WCAG 2.2 guarantees (focus ring,
target size, validity conveyed beyond color).

<Demo>
  <fluid-number-input value="42" aria-label="Quantity" style="max-width: 12rem;"></fluid-number-input>
</Demo>

## Examples

### Basic

The stepper buttons increment / decrement by `step`; the field also accepts
typed input and native <kbd>↑</kbd>/<kbd>↓</kbd>.

<Demo>
  <fluid-number-input value="42" aria-label="Quantity" style="max-width: 12rem;"></fluid-number-input>
</Demo>

<ConformanceCode code={`<fluid-number-input name="qty" value="42"></fluid-number-input>`} />

### Min, max, step

`min` / `max` clamp the value (and disable the matching stepper at the bound);
`step` sets the increment.

<Demo>
  <fluid-number-input value="50" min="0" max="100" step="5" aria-label="Percent" style="max-width: 12rem;"></fluid-number-input>
</Demo>

<ConformanceCode code={`<fluid-number-input name="percent" value="50" min="0" max="100" step="5"></fluid-number-input>`} />

### Without stepper buttons

`no-steppers` keeps the numeric validation + formatting but drops the buttons:
useful in tight layouts or when you only want the typed field.

<Demo>
  <fluid-number-input value="3.14159" step="0.00001" no-steppers aria-label="Pi" style="max-width: 12rem;"></fluid-number-input>
</Demo>

<ConformanceCode code={`<fluid-number-input value="3.14159" step="0.00001" no-steppers></fluid-number-input>`} />

## Listening for input

Like `<fluid-input>`, it fires **`fluid-input`** on every keystroke / stepper
click and **`fluid-change`** when the value is committed. Both carry the value
in `event.detail.value`.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-number-input id="qty" value="1" aria-label="Quantity"></fluid-number-input>

    <script type="module">
      document.getElementById("qty").addEventListener("fluid-change", (e) => {
        console.log("committed:", e.detail.value);
      });
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="Quantity.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/number-input";

    export function Quantity() {
      const ref = useRef<HTMLElement>(null);
      useEffect(() => {
        const el = ref.current!;
        const onChange = (e: Event) =>
          console.log((e as CustomEvent).detail.value);
        el.addEventListener("fluid-change", onChange);
        return () => el.removeEventListener("fluid-change", onChange);
      }, []);
      return <fluid-number-input ref={ref} value="1" aria-label="Quantity" />;
    }
    ```
  </TabItem>
  <TabItem label="Vue">
    ```vue title="Quantity.vue"
    <script setup lang="ts">

    const onChange = (e: CustomEvent) => console.log(e.detail.value);
    </script>

    ```
  </TabItem>

### `fluid-input` vs `fluid-change`

- **`fluid-input`**: every keystroke and every stepper click. Use it for live
  recalculation (running totals, previews).
- **`fluid-change`**: once the value is committed (blur or stepper click). Use
  it for "save when done" / persisting the value.

## Inside a form

Form-associated: `name` + `value` submit with the form, and the native
constraints (`min` / `max` / `step` / `required`) participate in validation.

## Theming

Every styled property reads a **component-scoped `--fluid-number-input-*` token
that falls back to a main semantic var** (the override ladder), including the
stepper buttons. The three layers (brand → component → instance) work the same
way as on [`<fluid-input>`](/components/input/#theming). The **complete list**
of both layers is in the [API tables below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-number-input-border-focus: #6366f1;
  --fluid-number-input-stepper-hover-bg: var(--fluid-surface-muted);
}
```

<Aside type="tip">
  Select a number input in the [Theme builder](/playground/) to tweak these
  tokens visually.
</Aside>

### Beyond tokens: `::part()`

The shell is [`part="base"`](#api), the field is `part="input"`, and the
steppers are `part="steppers"` / `part="stepper-up"` / `part="stepper-down"`.
Reach any of them with [`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part)
for CSS not covered by a token:

```css
/* Wider, pill-cornered steppers. */
fluid-number-input::part(steppers) { width: 2.25rem; }
fluid-number-input::part(base) { border-radius: 12px; }
```

## When to use

- Quantities, ages, counts, percentages: anything with a numeric range where
  the exact value matters.

### When *not* to use

- **Relative position** matters more than the exact number: use
  [`<fluid-slider>`](/components/slider/).
- **Free-form text that happens to contain digits** (phone, postcode): use
  [`<fluid-input>`](/components/input/) with the right `type`.

## Accessibility

Wraps a native `<input type="number">`, so it inherits the spinbutton role,
numeric keyboard, and constraint validation. The notes below are
number-input-specific; the cross-component contract is in the
[Accessibility guide](/guides/accessibility/).

### Keyboard

| Key | Action |
| --- | --- |
| <kbd>Tab</kbd> | Move focus to/from the field. |
| <kbd>↑</kbd> / <kbd>↓</kbd> | Step the value up/down (native). |
| Typing | Enter a value directly. |

The stepper buttons are **mouse/touch affordances**, they're `tabindex="-1"`
(keyboard users step with the arrow keys on the field), so there's no redundant
tab stop. The focus ring reads `--fluid-focus-ring-width`.

### Names

Provide an accessible name with `aria-label`, or associate a visible `<label>`
([SC 4.1.2](https://www.w3.org/WAI/WCAG22/Understanding/name-role-value)). The
steppers carry their own `aria-label`s ("Increase" / "Decrease").

### State semantics

- `disabled` / `readonly` behave like the native input; steppers disable at the
  `min` / `max` bounds.
- The invalid state sets **`aria-invalid="true"`** and colors the border with
  the **danger tone**, never by color alone
  ([SC 1.4.1](https://www.w3.org/WAI/WCAG22/Understanding/use-of-color)); pair
  it with a message + `aria-describedby`.

### Motion & target size

- Transitions honor `prefers-reduced-motion`.
- The field meets the AA 24×24 target floor.

### AA vs AAA

Ships **WCAG 2.2 AA**; flip the **Conformance** toggle at the top to **AAA** and
the field + snippets update. It reads conformance tokens, so it scales with no
per-component branching:

| Token | AA | AAA | Criterion |
| --- | --- | --- | --- |
| `--fluid-target-min` | 24px | **44px** | 2.5.8 → [2.5.5 Target Size (Enhanced)](https://www.w3.org/WAI/WCAG22/Understanding/target-size-enhanced) |
| `--fluid-focus-ring-width` | 2px | **3px** | 2.4.7 → [2.4.13 Focus Appearance](https://www.w3.org/WAI/WCAG22/Understanding/focus-appearance) |
