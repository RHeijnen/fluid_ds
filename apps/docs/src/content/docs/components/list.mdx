`<fluid-list>` is a styled `role="list"` surface; each `<fluid-list-item>` is a
`role="listitem"` row with four regions: a **leading** slot (icon / avatar), the
default slot (primary text), a **description** slot (secondary text), and a
**trailing** slot (meta / actions). A row is passive by default. Set
`interactive` to make it a real `<button>` that fires **`fluid-select`**, or set
`href` to make it a real `<a>`.

### Interactive rows

Add `interactive` to render the row body as a `<button>`. It gains hover, a
focus ring, and fires **`fluid-select`** on click, Enter, or Space. `disabled`
makes the row inert.

## Listening for selection

An `interactive` row fires **`fluid-select`** when it is activated. The event
bubbles, so you can listen on the parent `<fluid-list>`.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-list id="ws">
      <fluid-list-item interactive>Acme Corp</fluid-list-item>
      <fluid-list-item interactive>Side Project</fluid-list-item>
    </fluid-list>

    <script type="module">
      document.getElementById("ws").addEventListener("fluid-select", (e) => {
        console.log("picked", e.target.textContent.trim());
      });
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="Workspaces.tsx"
    import "@fluid-ds/components/define/list";

    export function Workspaces() {
      return (
        <fluid-list label="Workspaces"
          onfluid-select={(e) => console.log(e.target.textContent.trim())}>
          <fluid-list-item interactive>Acme Corp</fluid-list-item>
          <fluid-list-item interactive>Side Project</fluid-list-item>
        </fluid-list>
      );
    }
    ```
  </TabItem>
  <TabItem label="Vue">
    ```vue title="Workspaces.vue"
    <script setup lang="ts">
    import "@fluid-ds/components/define/list";
    function pick(e: Event) {
      console.log((e.target as HTMLElement).textContent?.trim());
    }
    </script>

    <template>
      <fluid-list label="Workspaces" @fluid-select="pick">
        <fluid-list-item interactive>Acme Corp</fluid-list-item>
        <fluid-list-item interactive>Side Project</fluid-list-item>
      </fluid-list>
    </template>
    ```
  </TabItem>
  <TabItem label="Angular">
    ```ts title="workspaces.component.ts"
    import { Component, CUSTOM_ELEMENTS_SCHEMA } from "@angular/core";
    import "@fluid-ds/components/define/list";

    @Component({
      standalone: true,
      schemas: [CUSTOM_ELEMENTS_SCHEMA],
      selector: "app-workspaces",
      template: `
        <fluid-list label="Workspaces" (fluid-select)="pick($event)">
          <fluid-list-item interactive>Acme Corp</fluid-list-item>
        </fluid-list>
      `
    })
    export class Workspaces {
      pick(e: Event) { console.log((e.target as HTMLElement).textContent?.trim()); }
    }
    ```
  </TabItem>

## When to use

- Settings rows, file lists, member lists, navigation menus.
- A scannable column of items where each row carries an icon, a title, a
  secondary line, and trailing meta.
- Use `interactive` rows for "pick one to drill in" patterns, and `href` rows
  for navigation.

For a free-form surface of grouped content, reach for
[`<fluid-card>`](/components/card/). For a menu of actions tied to a trigger,
use [`<fluid-menu>`](/components/menu/).

## Theming

Every styled property reads a **component-scoped token that falls back to a main
semantic var** (the override ladder). The list owns `--fluid-list-*` (surface,
border, divider) and each row owns `--fluid-list-item-*` (background, hover,
text, secondary text, radius, padding). The **complete list** is in the
[API tables below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-list-bg: var(--fluid-surface-base);
  --fluid-list-divider: var(--fluid-border-default);
  --fluid-list-item-hover-bg: var(--fluid-surface-muted);
  --fluid-list-item-secondary-fg: var(--fluid-text-secondary);
}
```

### Beyond tokens: `::part()`

The list surface is [`part="base"`](#api). Each row exposes
`part="base"` plus `leading`, `content`, `primary`, `description`, and
`trailing`. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

- The list surface is `role="list"`; each row is `role="listitem"`, so assistive
  tech announces position ("3 of 5").
- Give the list an accessible name with `label` (or `aria-label`) when the
  surrounding context does not already name it.
- `interactive` rows wrap a native `<button>` and link rows wrap a native `<a>`,
  so they are in the tab order and inherit the platform keyboard contract: a
  button activates on Enter and Space, a link on Enter.
- The focus ring reads `--fluid-focus-ring-width` / `--fluid-focus-ring-color`
  and never relies on color alone.
- Interactive rows meet the minimum target size: the row is at least 2.25rem
  tall, and scales to 44×44 when an ancestor opts into AAA via
  `data-fluid-conformance="aaa"` through `--fluid-target-min`.

### AA vs AAA

- **AA (default):** rows meet the 24×24 minimum target size and a 3:1 focus-ring
  contrast.
- **AAA:** with `data-fluid-conformance="aaa"` set on an ancestor, interactive
  rows grow to the enhanced 44×44 target and the focus ring widens, with no
  per-instance changes.
