`<fluid-mosaic>` is a bento-style grid: tiles of mixed sizes pack tightly into a
uniform column track with dense auto-flow, so smaller tiles backfill the gaps
larger ones leave. Columns are **intrinsic** (auto-fill) by default or **fixed**
via `cols`; each tile's height comes from how many rows it spans. Give
[`<fluid-mosaic-item>`](#fluid-mosaic-item) children a `size` preset or explicit
spans.

## Examples

### Size presets

Each tile takes a `size`: `normal` (1×1, default), `wide` (2×1), `tall` (1×2),
or `large` (2×2).

```html
<fluid-mosaic cols="4" row-height="7rem">
  <fluid-mosaic-item size="large">…</fluid-mosaic-item>
  <fluid-mosaic-item size="wide">…</fluid-mosaic-item>
  <fluid-mosaic-item>…</fluid-mosaic-item>
  <fluid-mosaic-item size="tall">…</fluid-mosaic-item>
</fluid-mosaic>
```

### Intrinsic columns

Drop `cols` for an auto-filling track that reflows with the container, set
`min-col-width` to tune the breakpoint.

```html
<fluid-mosaic min-col-width="10rem" row-height="8rem">
  <fluid-mosaic-item size="wide">…</fluid-mosaic-item>
  <fluid-mosaic-item>…</fluid-mosaic-item>
</fluid-mosaic>
```

### Explicit spans

`col-span` and `row-span` override the `size` preset on their axis.

```html
<fluid-mosaic cols="4" row-height="6rem">
  <fluid-mosaic-item col-span="3">banner</fluid-mosaic-item>
  <fluid-mosaic-item row-span="2">rail</fluid-mosaic-item>
</fluid-mosaic>
```

## When to use

- **Dashboards & galleries** where tiles have deliberately different weights.
- **Marketing bento grids**: a feature highlight beside smaller stat tiles.

### When *not* to use

- **Uniform cells** that should all match: use
  [`<fluid-grid>`](/components/grid/).
- **A flowing list of equal cards**: a [`<fluid-grid>`](/components/grid/) in
  intrinsic mode is simpler.

## Theming

Every layout knob reads a **component-scoped `--fluid-mosaic-*` token that falls
back to a main semantic var** (the override ladder); the `gap` / `min-col-width`
/ `row-height` attributes set the matching token per instance. The **complete
list** is in the [API table below](#api).

```css title="brand.css"
fluid-mosaic {
  --fluid-mosaic-gap: var(--fluid-space-3);
  --fluid-mosaic-row-height: 9rem;
  --fluid-mosaic-min-col: 14rem;
}
```

<Aside type="note">
  Tiles are structural only, they clip overflow and span the track, but carry
  no background or radius. Style the content you slot in (e.g. a
  [`<fluid-card>`](/components/card/)) to taste.
</Aside>

## Accessibility

`<fluid-mosaic>` and `<fluid-mosaic-item>` are **presentational layout
primitives**, no ARIA roles, no change to the accessibility tree; slotted
content keeps its own semantics.

- **Dense packing reorders visually, not in the DOM.** `grid-auto-flow: dense`
  can place a later tile before an earlier one on screen, so keep the DOM order
  matching the intended reading/tab order
  ([SC 1.3.2](https://www.w3.org/WAI/WCAG22/Understanding/meaningful-sequence),
  [SC 2.4.3](https://www.w3.org/WAI/WCAG22/Understanding/focus-order)). Don't
  rely on `dense` to convey meaning.
- **Reflow.** Intrinsic columns and a min-width-capped track satisfy
  [SC 1.4.10 Reflow](https://www.w3.org/WAI/WCAG22/Understanding/reflow).
