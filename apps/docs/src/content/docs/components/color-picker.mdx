`<fluid-color-picker>` is a single field with three input paths: click the left
swatch to open the OS color picker, type a hex value in the field, or click one
of the optional preset swatches. The swatch is slotted into the `prefix` of an
internal [`<fluid-input>`](/components/input/), so it reads as one unified
control. Form-associated; validates hex inline.

<Demo>
  <fluid-color-picker
    value="#0ea5e9"
    aria-label="Brand"
    palette='["#0ea5e9","#6366f1","#a855f7","#ec4899","#f43f5e","#f97316","#facc15","#22c55e"]'
  ></fluid-color-picker>
</Demo>

## Inside a form

The picker is form-associated: it submits `name=<hex>`, enforces `#rgb` /
`#rrggbb`, and trips `patternMismatch` on anything else.

```html
<form>
  <fluid-color-picker name="accent" value="#6366f1" required></fluid-color-picker>
  <fluid-button type="submit">Save</fluid-button>
</form>
```

## Listening for changes

`fluid-input` fires on every change (typing, picker dragging, preset clicks);
`fluid-change` fires only when the value commits (blur, picker close). Both
carry `{ value }` in `detail`.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-color-picker id="accent" aria-label="Accent"></fluid-color-picker>

    <script type="module">
      const el = document.getElementById("accent");
      el.addEventListener("fluid-input", (e) => {
        document.body.style.background = e.detail.value;
      });
      el.addEventListener("fluid-change", (e) => console.log("committed:", e.detail.value));
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="AccentPicker.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/color-picker";

    export function AccentPicker() {
      const ref = useRef<HTMLElement>(null);
      useEffect(() => {
        const el = ref.current!;
        const onInput = (e: Event) =>
          console.log((e as CustomEvent).detail.value);
        el.addEventListener("fluid-input", onInput);
        return () => el.removeEventListener("fluid-input", onInput);
      }, []);
      return <fluid-color-picker ref={ref} aria-label="Accent" />;
    }
    ```
  </TabItem>

## When to use

- Theme builders, brand customizers: anything that captures a single color
  value.
- A small set of brand colors users should mostly pick from: provide a
  `palette`, and the open-the-picker fallback handles edge cases.

### When *not* to use

- **Multi-color inputs** (gradients): build the composite from this component
  plus separate inputs for stops.

## Theming

Every styled property of the picker reads a **component-scoped
`--fluid-color-picker-*` token that falls back to a main semantic var** (the
override ladder). The hex field itself is a composed
[`<fluid-input>`](/components/input/), so its `--fluid-input-*` tokens (border,
background, focus ring) apply too. The **complete list** is in the
[API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-color-picker-swatch-size: 2.5rem;
  --fluid-color-picker-preset-size: 1.25rem;
  --fluid-color-picker-focus-ring: var(--fluid-focus-ring-color);
}
```

<Aside type="tip">
  Select a `fluid-color-picker` in the [Theme builder](/playground/) to tweak
  the swatch, preset, and focus tokens visually.
</Aside>

### Beyond tokens: `::part()`

The container is [`part="base"`](#api), the swatch is `part="swatch"`, the hex
field is `part="input"`, and the palette is `part="palette"`. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

The notes below are color-picker-specific; the cross-component contract is in
the [Accessibility guide](/guides/accessibility/).

### Names & roles

- The hex field carries its own accessible name ("Color hex" by default; set
  `aria-label` to override).
- The swatch is a real `<button>` with its own `aria-label` ("Color: open
  picker").
- The native `<input type="color">` is visually hidden and out of the tab
  order; the swatch button receives focus.
- The preset palette is a `role="listbox"` with `role="option"` children and
  `aria-selected` reflecting the current value.

### State semantics

Invalid hex values set `patternMismatch` via the constraint-validation API, so
submitting the parent form surfaces the browser's prompt
([SC 3.3.1](https://www.w3.org/WAI/WCAG22/Understanding/error-identification)).

### Motion & target size

- The swatch hover transition honors `prefers-reduced-motion`.
- The swatch fills the full field height (≥ **24/44px**), and each preset chip
  floors its box to `--fluid-target-min`, so AAA grows the chips to 44px
  without any per-size override.

### AA vs AAA

Ships **WCAG 2.2 AA**; flip the **Conformance** toggle to **AAA**:

| Token | AA | AAA | Criterion |
| --- | --- | --- | --- |
| `--fluid-target-min` | 24px | **44px** | 2.5.8 → [2.5.5 Target Size (Enhanced)](https://www.w3.org/WAI/WCAG22/Understanding/target-size-enhanced) |
| `--fluid-focus-ring-width` | 2px | **3px** | 2.4.7 → [2.4.13 Focus Appearance](https://www.w3.org/WAI/WCAG22/Understanding/focus-appearance) |
