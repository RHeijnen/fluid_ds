`<fluid-stack>` is the one-dimensional flow layout (flexbox), the 1D complement
to [`<fluid-grid>`](/components/grid/). Vertical by default for an evenly-spaced
**stack**; switch to `direction="horizontal"` with `wrap` for a **cluster**, a
row of chips, tags, or buttons that wraps onto new lines while keeping the gap
uniform.

<Demo>
  <fluid-stack direction="horizontal" wrap gap="0.5rem" style="max-width: 26rem;">
    <fluid-tag>Design</fluid-tag>
    <fluid-tag>Engineering</fluid-tag>
    <fluid-tag>Marketing</fluid-tag>
    <fluid-tag>Sales</fluid-tag>
    <fluid-tag>Support</fluid-tag>
    <fluid-tag>Finance</fluid-tag>
  </fluid-stack>
</Demo>

## Examples

### Vertical stack

The default, items flow down a column with an even `gap`.

    <fluid-input placeholder="Email" aria-label="Email"></fluid-input>
    <fluid-button>Continue</fluid-button>
  </fluid-stack>
</Demo>

```html
<fluid-stack gap="0.75rem">
  <fluid-input placeholder="Name"></fluid-input>
  <fluid-input placeholder="Email"></fluid-input>
```


### Cluster (wrapping)

`direction="horizontal"` + `wrap` keeps a uniform gap as items reflow onto new
lines, ideal for tags, filters, or button groups.

```html
<fluid-stack direction="horizontal" wrap gap="0.5rem">
  <fluid-tag>Design</fluid-tag>
  <fluid-tag>Engineering</fluid-tag>
  <fluid-tag>Marketing</fluid-tag>
</fluid-stack>
```

### Alignment & distribution

`align` is the cross axis; `justify` is the main axis. Both accept friendly
aliases (`start` · `center` · `end` · `between` · `around` · `evenly` ·
`stretch` · `baseline`).

<Demo>
  <fluid-stack direction="horizontal" justify="between" align="center" style="width: 24rem; border: 1px dashed var(--fluid-border-default); border-radius: var(--fluid-radius-md); padding: 0.75rem;">
    <span>Left</span>
    <fluid-button size="sm">Right</fluid-button>
  </fluid-stack>
</Demo>

```html
<fluid-stack direction="horizontal" justify="between" align="center">
  <span>Left</span>
  <fluid-button>Right</fluid-button>
</fluid-stack>
```

## When to use

- **Forms, toolbars, button rows, tag/chip groups**: anything that flows along
  a single axis.
- A wrapping group of items (**cluster**): `direction="horizontal"` + `wrap`.

### When *not* to use

- **Two-dimensional, uniform cells**: use [`<fluid-grid>`](/components/grid/).
- **Mixed-size tiles that pack densely**: use
  [`<fluid-mosaic>`](/components/mosaic/).

## Theming

Every layout knob reads a **component-scoped `--fluid-stack-*` token that falls
back to a main semantic var** (the override ladder); the `gap` / `align` /
`justify` attributes set the matching token per instance. The **complete list**
is in the [API table below](#api).

```css title="brand.css"
fluid-stack {
  --fluid-stack-gap: var(--fluid-space-6);
}
```

## Accessibility

`<fluid-stack>` is a **presentational layout primitive**, it adds no ARIA roles
and doesn't alter the accessibility tree, so slotted content keeps its own
semantics.

- **Source order = reading order.** Flow placement is visual only; keep the DOM
  order matching the intended reading/tab order
  ([SC 1.3.2](https://www.w3.org/WAI/WCAG22/Understanding/meaningful-sequence),
  [SC 2.4.3](https://www.w3.org/WAI/WCAG22/Understanding/focus-order)).
- **Reflow.** `wrap` lets a horizontal cluster collapse onto new lines instead
  of forcing horizontal scroll
  ([SC 1.4.10 Reflow](https://www.w3.org/WAI/WCAG22/Understanding/reflow)).
