`<fluid-badge>` is the small, tinted pill that sits beside something else, a
"new" marker, a count next to an icon, a status label on a row. Five variants
signal severity: `neutral` (default), `info`, `success`, `warning`, `danger`.
Set `dot` for a tiny circular indicator with no text.

<Demo>
  <fluid-badge variant="success">Active</fluid-badge>
  <fluid-badge variant="danger">3</fluid-badge>
  <fluid-badge variant="warning" dot></fluid-badge>
</Demo>

## When to use

- Status labels: "Active", "Pending", "Failed".
- Counts next to icons or labels.
- Tiny inline indicators (the dot mode).

Reach for [`<fluid-tag>`](/components/tag/) when the label is a piece of
user-supplied data the user can remove (a filter chip, a recipient pill).
Tags are interactive; badges are not.

## Theming

Every styled property reads a **component-scoped `--fluid-badge-*` token that
falls back to a main semantic var or color primitive** (the override ladder);
each variant has its own `bg` / `fg` pair. The **complete list** is in the
[API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-badge-bg: var(--fluid-color-neutral-200);
  --fluid-badge-fg: var(--fluid-color-neutral-800);
  --fluid-badge-success-bg: var(--fluid-color-emerald-100);
  --fluid-badge-success-fg: var(--fluid-color-emerald-800);
  --fluid-badge-danger-bg: var(--fluid-color-red-100);
  --fluid-badge-danger-fg: var(--fluid-color-red-800);
}
```

### Beyond tokens: `::part()`

The pill is [`part="base"`](#api). Reach it with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS not
covered by a token.

## Accessibility

- The badge is a passive label: no implicit ARIA role. If the badge
  conveys a state critical to surrounding content (an unread count,
  a "live" indicator), wrap it in a parent with an `aria-label` that
  spells it out, or include explanatory text alongside it.
- The `dot` mode is purely decorative: the colored circle has no text
  alternative. Don't rely on color alone; pair it with a visible label
  or `aria-label` on the enclosing element.
