`<fluid-popover>` is the general-purpose floating panel, click a trigger to
open it, click outside or press <kbd>Escape</kbd> to close. Positioned with
floating-ui, so it auto-flips when it would overflow the viewport.

<Demo>
  <fluid-popover>
    <fluid-button slot="trigger">Open popover</fluid-button>
    <p style="margin: 0;">Any content goes here.</p>
  </fluid-popover>
</Demo>

### Placement

```html
<fluid-popover placement="top-end">…</fluid-popover>
```

Accepts every floating-ui placement: `top`, `top-start`, `top-end`,
`bottom`, `bottom-start` (default), `bottom-end`, `left`, `left-start`,
`right`, `right-start`, etc.

### Distance & skidding

`distance` is gap from the trigger; `skidding` shifts along the cross
axis.

```html
<fluid-popover placement="bottom" distance="12" skidding="-8">…</fluid-popover>
```

### Settings panel

A common use, an icon-only button that reveals a small form.

```html
<fluid-popover placement="bottom-end">
  <fluid-button slot="trigger" variant="ghost" aria-label="Settings">
    <fluid-icon name="settings"></fluid-icon>
  </fluid-button>
  <form>
    <fluid-switch>Dark mode</fluid-switch>
    <fluid-switch>Compact density</fluid-switch>
  </form>
</fluid-popover>
```

## Listening for changes

`<fluid-popover>` fires **`fluid-show`** and **`fluid-hide`**, and exposes
imperative `show()` / `hide()` / `toggle()`.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-popover id="p"><fluid-button slot="trigger">Open</fluid-button>…</fluid-popover>

    <script type="module">
      const p = document.getElementById("p");
      p.addEventListener("fluid-show", () => console.log("opened"));
      p.addEventListener("fluid-hide", () => console.log("closed"));
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="Settings.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/popover";

    export function Settings() {
      const ref = useRef<HTMLElement>(null);
      useEffect(() => {
        const el = ref.current!;
        const onShow = () => console.log("opened");
        el.addEventListener("fluid-show", onShow);
        return () => el.removeEventListener("fluid-show", onShow);
      }, []);
      return <fluid-popover ref={ref}><button slot="trigger">Open</button>…</fluid-popover>;
    }
    ```
  </TabItem>
  <TabItem label="Vue">
    ```vue title="Settings.vue"
    <script setup lang="ts">
    import "@fluid-ds/components/define/popover";
    </script>

    <template>
      <fluid-popover @fluid-show="() => {}" @fluid-hide="() => {}">
        <button slot="trigger">Open</button>…
      </fluid-popover>
    </template>
    ```
  </TabItem>
  <TabItem label="Angular">

## When to use

- Contextual options or controls attached to a specific element.

### When *not* to use

- **A list of actions**: use [`<fluid-dropdown>`](/components/dropdown/).
- **Text-only help**: use [`<fluid-tooltip>`](/components/tooltip/) (hover-
  triggered, lighter).
- **A flow that must be completed/submitted first**: use
  [`<fluid-dialog>`](/components/dialog/) (modal, not dismissible).

## Theming

Every styled property reads a **component-scoped `--fluid-popover-*` token that
falls back to a main semantic var** (the override ladder). The **complete list**
is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-popover-bg: var(--fluid-surface-base);
  --fluid-popover-border: var(--fluid-border-default);
  --fluid-popover-radius: var(--fluid-radius-md);
}
```

### Beyond tokens: `::part()`

The container exposes [`part="base"`](#api) and `part="panel"`. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

The cross-component contract is in the
[Accessibility guide](/guides/accessibility/); the notes below are
popover-specific.

### Focus & keyboard

- The panel is `role="dialog"`; the trigger gets `aria-haspopup="true"` +
  `aria-expanded`.
- Focus moves into the first focusable element on open; on close it returns to
  the trigger ([SC 2.4.3](https://www.w3.org/WAI/WCAG22/Understanding/focus-order)).
- <kbd>Escape</kbd> and outside-click both dismiss.
- For a form that **must be submitted before the user moves on**, use
  [`<fluid-dialog>`](/components/dialog/), popovers are dismissible.

### Motion

The open/close scale + fade honors `prefers-reduced-motion`.
