`<fluid-tooltip>` is the small contextual hint, visible on hover or focus of
its slotted child, dismissed when the pointer leaves or focus moves on.
Positioning is floating-ui-powered, so it auto-flips when it would overflow the
viewport. Use it for **short, supplementary** text; for anything more
substantial, reach for [`<fluid-popover>`](/components/popover/).

<Demo>
  <fluid-tooltip content="Save your changes (⌘S)">
    <fluid-button>Save</fluid-button>
  </fluid-tooltip>
</Demo>


### Rich content

Slot anything into the `content` slot for richer tooltips. It overrides
the `content` attribute.

```html
<fluid-tooltip>
  <fluid-button>Hover me</fluid-button>
  <div slot="content">
    <strong>Pro tip</strong><br/>
    Hold <kbd>Shift</kbd> to do it without confirmation.
  </div>
</fluid-tooltip>
```

### Delays

`show-delay` and `hide-delay` (ms) help avoid flashing tooltips during
quick pointer passes.

```html
<fluid-tooltip content="…" show-delay="200" hide-delay="100">…</fluid-tooltip>
```

### Programmatic open

```ts
const tip = document.querySelector("fluid-tooltip");
tip.open = true;     // forced visible
tip.open = false;    // back to hover/focus
```

## When to use

- A short hint that explains an icon-only button, supplements a label,
  or provides keyboard shortcuts.
- **Don't** put critical information in a tooltip: touch-only users
  can't hover, and screen readers can announce tooltips inconsistently.
- For richer panels with form controls or links, use
  [`<fluid-popover>`](/components/popover/).

## Theming

Every styled property reads a **component-scoped `--fluid-tooltip-*` token that
falls back to a main semantic var** (the override ladder). The **complete list**
is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-tooltip-bg: var(--fluid-color-neutral-900);
  --fluid-tooltip-color: var(--fluid-color-white);
  --fluid-tooltip-radius: 8px;
}
```

### Beyond tokens: `::part()`

The floating element is [`part="popover"`](#api) (and `part="base"` wraps the
trigger). Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

The cross-component contract is in the
[Accessibility guide](/guides/accessibility/); the notes below are
tooltip-specific (WAI-ARIA tooltip pattern).

### Names & roles

- The popover is `role="tooltip"`; the slotted trigger gets `aria-describedby`
  pointing at it, so the text is announced after the trigger's accessible name
  ([SC 4.1.2](https://www.w3.org/WAI/WCAG22/Understanding/name-role-value)).

### Keyboard & pointer (SC 1.4.13)

- Shows on **hover and focus**, so keyboard users see it on <kbd>Tab</kbd>.
- <kbd>Escape</kbd> dismisses it (dismissible).
- It stays visible while hovered and doesn't steal pointer events
  (`pointer-events: none`), satisfying
  [SC 1.4.13 Content on Hover or Focus](https://www.w3.org/WAI/WCAG22/Understanding/content-on-hover-or-focus).

### Don't hide essential info here

A tooltip is supplementary, the trigger must still make sense on its own, and
never put the only copy of critical information (or any interactive control)
inside one.

### Motion

The fade honors `prefers-reduced-motion`.
