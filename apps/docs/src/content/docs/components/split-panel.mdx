`<fluid-split-panel>` lays out two slotted panels (`start` and `end`)
with a draggable divider between them. The divider responds to pointer
drag, touch drag, and arrow keys when focused. Position is a percentage
from the start edge, so it scales as the parent resizes.

    <div slot="end" style="padding: var(--fluid-space-3); height: 100%;">Right pane</div>
  </fluid-split-panel>
</Demo>

## Listening for resize

The divider fires **`fluid-reposition`** whenever the split changes (drag or
keyboard), with the new percentage on `event.detail.position`.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-split-panel id="split">
      <div slot="start">A</div>
      <div slot="end">B</div>
    </fluid-split-panel>

    <script type="module">
      document.getElementById("split").addEventListener("fluid-reposition", (e) => {
        console.log("divider at", e.detail.position, "%");
      });
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="Editor.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/split-panel";

    export function Editor() {
      const ref = useRef<HTMLElement>(null);
      useEffect(() => {
        const el = ref.current!;
        const onMove = (e: any) => console.log(e.detail.position);
        el.addEventListener("fluid-reposition", onMove);
        return () => el.removeEventListener("fluid-reposition", onMove);
      }, []);
      return (
        <fluid-split-panel ref={ref}>
          <div slot="start">A</div>
          <div slot="end">B</div>
        </fluid-split-panel>
      );
    }
    ```
  </TabItem>
  <TabItem label="Vue">

## When to use

- Editor layouts (file tree + editor; preview + source).
- Master-detail layouts where the user wants to resize the columns.
- For app-level shells (header, nav, main, aside, footer), use
  [`<fluid-page>`](/components/page/), it covers the same layout
  shapes without resizing.

## Theming

The divider reads **component-scoped `--fluid-split-*` tokens that fall back to
main semantic vars** (the override ladder); its active/focus colors come off
the accent track. The **complete list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-split-divider-size: 4px;
  --fluid-split-divider-color: var(--fluid-border-default);
  --fluid-split-divider-active-color: var(--fluid-accent-base);
}
```

### Beyond tokens: `::part()`

The split exposes [`part="base"`](#api) plus `part="panel"` and
`part="divider"`. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token, e.g. a grip texture on the divider.

## Accessibility

- The divider carries `role="separator"` with `aria-orientation`
  matching the split, `aria-valuemin="0"`, `aria-valuemax="100"`, and
  `aria-valuenow` reflecting the current position.
- Keyboard, when the divider is focused:
  - <kbd>←</kbd>/<kbd>→</kbd> (horizontal) or <kbd>↑</kbd>/<kbd>↓</kbd>
    (vertical) move 1%.
  - <kbd>Shift</kbd>+arrow moves 10%.
  - <kbd>Home</kbd>/<kbd>End</kbd> snap to 0% / 100%.
- The divider has a visible focus ring; the panels themselves are
  ordinary regions you should label appropriately.
