`<fluid-result>` is the full-width status block for the end of a flow: a large
status icon, a `title`, an optional `subtitle`, extra detail, and a row of
actions, centered in a column. It is the bigger sibling of
[`<fluid-empty-state>`](/components/empty-state/), built for confirmation pages,
error screens, and full-page "not found" views.

The status reads from a text **title** (never color alone) plus a status-tinted
icon drawn from the theme-independent semantic tokens, so it stays meaningful in
every brand. Success / info / 404 announce politely (`role="status"`); error /
warning announce assertively (`role="alert"`).

## Slots

- **`icon`** the status glyph. Defaults to a status-appropriate icon.
- **default** extra detail rendered below the subtitle.
- **`actions`** call-to-action buttons below the content.

## When to use

- Confirmation pages after a checkout, sign-up, or submission.
- Full-page error screens (request failed, access denied).
- "Not found" / 404 routes.

Reach for [`<fluid-empty-state>`](/components/empty-state/) for the lighter
"nothing here yet" placeholder inside a list or panel, and
[`<fluid-callout>`](/components/callout/) for an inline notice that sits in the
page flow rather than taking over the view.

## Theming

Every styled property reads a **component-scoped `--fluid-result-*` token that
falls back to a main semantic var** (the override ladder). The status icon
colors fall back to the theme-independent semantic status tokens
(`--fluid-success-base`, `--fluid-danger-base`, `--fluid-info-base`,
`--fluid-warning-base`), so they stay meaningful across brands. The **complete
list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-result-fg: var(--fluid-text-primary);
  --fluid-result-muted-fg: var(--fluid-text-secondary);
  --fluid-result-icon-size: 5rem;
  --fluid-result-success-icon: var(--fluid-success-base);
}
```

### Beyond tokens: `::part()`

The column is [`part="base"`](#api) with `part="icon"`, `part="title"`,
`part="subtitle"`, `part="content"`, and `part="actions"`. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

- The block is a live region: `role="status"` (polite) for success / info / 404,
  and `role="alert"` (assertive) for error / warning, so a result rendered after
  a user action is announced. `aria-atomic="true"` reads the title and subtitle
  together.
- Status is never conveyed by color alone (SC 1.4.1): the `title` carries the
  text, and the icon is a redundant cue.
- The default status icons are decorative (`aria-hidden`), the title supplies the
  meaning. If you replace the icon with a meaningful one, give it its own label.
- Action buttons in the `actions` slot are real focusable controls; keep their
  labels clear (for example "Retry", not "OK").
- Honors `prefers-reduced-motion`.
