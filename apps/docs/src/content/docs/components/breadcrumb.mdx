`<fluid-breadcrumb>` is the trail of links that shows where in the hierarchy the
user is. Items are `<fluid-breadcrumb-item>` elements, each renders as a link
when `href` is set, or as plain text for the current page. The breadcrumb
auto-marks the last visible item as `current` and hides its trailing separator.

<Demo>
  <fluid-breadcrumb>
    <fluid-breadcrumb-item href="/">Home</fluid-breadcrumb-item>
    <fluid-breadcrumb-item href="/components/">Components</fluid-breadcrumb-item>
    <fluid-breadcrumb-item>Breadcrumb</fluid-breadcrumb-item>
  </fluid-breadcrumb>
</Demo>

The last item is automatically marked as the current page (it gets
`aria-current="page"` and renders without a link).

### With icons

Use the `prefix` slot on any item to add a leading icon.

<Demo>
  <fluid-breadcrumb>
    <fluid-breadcrumb-item href="/">
      <fluid-icon slot="prefix" name="home"></fluid-icon>
      Home
    </fluid-breadcrumb-item>
    <fluid-breadcrumb-item href="/settings/">Settings</fluid-breadcrumb-item>
    <fluid-breadcrumb-item>Profile</fluid-breadcrumb-item>
  </fluid-breadcrumb>
</Demo>

## When to use

- Pages that live more than one level deep in a hierarchy.
- Anywhere "you are here" context helps the user orient and back out.

For a deeper navigation tree (folders inside folders), use
[`<fluid-tree>`](/components/tree/). For a flat back-and-forth between
peer pages, use [`<fluid-tabs>`](/components/tabs/).

## Theming

Every styled property reads a **component-scoped token that falls back to a main
semantic var** (the override ladder): the trail exposes `--fluid-breadcrumb-gap`
and each item exposes `--fluid-breadcrumb-item-*`. The **complete list** is in
the [API tables below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-breadcrumb-item-fg: var(--fluid-text-secondary);
  --fluid-breadcrumb-item-current-fg: var(--fluid-text-primary);
  --fluid-breadcrumb-item-accent: var(--fluid-accent-base);
}
```

### Beyond tokens: `::part()`

The item exposes [`part="base"`](#api), `part="label"`, and `part="separator"`.
Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

The cross-component contract is in the
[Accessibility guide](/guides/accessibility/); the notes below are
breadcrumb-specific.

### Names & roles

- The host renders as `<nav>` with `aria-label="Breadcrumb"` by default
  (override with your own `aria-label`).
- The current item gets `aria-current="page"`.
- Items with `href` are real anchors: right-click, middle-click, and
  copy-link-address all work.
- The separator is `aria-hidden="true"` so screen readers don't read "slash"
  between every link.

### Target size

Breadcrumb links are **inline text** within the trail, so they fall under the
WCAG [target-size *inline* exception](https://www.w3.org/WAI/WCAG22/Understanding/target-size-minimum)
(2.5.8), they aren't floored to `--fluid-target-min`. The focus ring still
honors `--fluid-focus-ring-width` (3px at AAA).
