Clamps slotted text to a fixed number of lines and reveals the rest behind a
disclosure toggle. The toggle is a real `<button>` and only renders when the
content genuinely overflows, measured from the rendered box, so a short string
shows no control at all.

<Demo>
  <div style="max-width: 28rem;">
    <fluid-truncate lines="3">
      Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod
      tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim
      veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea
      commodo consequat. Duis aute irure dolor in reprehenderit in voluptate
      velit esse cillum dolore eu fugiat nulla pariatur.
    </fluid-truncate>
  </div>
</Demo>

## Listening for fluid-toggle

The component fires **`fluid-toggle`** whenever the user expands or collapses,
with `detail.expanded` carrying the new state.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-truncate id="t">A long passage of text…</fluid-truncate>

    <script type="module">
      document.getElementById("t").addEventListener("fluid-toggle", (e) => {
        console.log("expanded:", e.detail.expanded);
      });
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="Bio.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/truncate";

    export function Bio({ text }: { text: string }) {
      const ref = useRef<HTMLElement>(null);
      useEffect(() => {
        const el = ref.current;
        const onToggle = (e: any) => console.log(e.detail.expanded);
        el?.addEventListener("fluid-toggle", onToggle);
        return () => el?.removeEventListener("fluid-toggle", onToggle);
      }, []);
      return <fluid-truncate ref={ref} lines={3}>{text}</fluid-truncate>;
    }
    ```
  </TabItem>
  <TabItem label="Vue">
    ```vue title="Bio.vue"
    <script setup lang="ts">
    import "@fluid-ds/components/define/truncate";
    defineProps<{ text: string }>();
    </script>

    <template>
      <fluid-truncate :lines="3" @fluid-toggle="(e) => console.log(e.detail.expanded)">
        {{ text }}
      </fluid-truncate>
    </template>
    ```
  </TabItem>
  <TabItem label="Angular">

## Theming

Every styled property reads a **component-scoped `--fluid-truncate-*` token that
falls back to a main semantic var** (the override ladder), giving brand →
component → instance overrides for free. The only color the component owns is
the toggle text, `--fluid-truncate-toggle-fg`, which defaults to
`--fluid-accent-base`. The line count is set with the `lines` prop (it drives an
internal `--fluid-truncate-lines` style), not a hand-set token. The
**complete** variable list is in the [API table below](#api).

```css title="brand.css"
/* component-wide: recolor every toggle */
[data-fluid-brand="custom"] {
  --fluid-truncate-toggle-fg: var(--fluid-accent-base);
}

/* one instance */
[data-fluid-id="legal-copy"] {
  --fluid-truncate-toggle-fg: var(--fluid-text-secondary);
}
```

### Beyond tokens: `::part()`

The wrapper is [`part="base"`](#api), the clamped region is `part="content"`,
and the disclosure button is `part="toggle"`. Reach any of them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## When to use

- Long descriptions, bios, or reviews that should not dominate a card.
- Multi-line captions where the full text is occasionally useful but not always.
- Comment bodies in a dense feed.

### When *not* to use

- For collapsing arbitrary rich content (headings, media, forms), reach for an
  [accordion](/components/accordion/) or
  [details/disclosure](/components/details/) so the structure stays semantic.
- For a single line that should end in an ellipsis with no toggle, plain CSS
  `text-overflow: ellipsis` is simpler.

## Accessibility

The toggle follows the WAI-ARIA
[Disclosure pattern](https://www.w3.org/WAI/ARIA/apg/patterns/disclosure/): a
native `<button>` whose `aria-expanded` reflects the open state and whose
`aria-controls` points at the clamped region. The text itself is passive and
carries no role. See the
[accessibility guide](/guides/accessibility/) for the cross-component contract.

### Keyboard

| Key | Action |
| --- | --- |
| `Tab` | Moves focus to the show-more / show-less button (when present). |
| `Enter` / `Space` | Toggles the content open or closed. |

The button shows a `:focus-visible` ring built from the focus-ring tokens
(`--fluid-focus-ring-width`, `--fluid-focus-ring-offset`).

### Names

The button's accessible name is its visible label, `more-label` when collapsed,
`less-label` when expanded. Keep both labels meaningful in isolation; the
default "Show more" / "Show less" satisfy label-in-name.

### State semantics

`aria-expanded` flips synchronously with the `expanded` property, and
`aria-controls` references the `id` of the clamped region so assistive
technology can associate the trigger with the content it reveals.

### Motion & target size

The toggle's hover transition collapses under
`prefers-reduced-motion: reduce` (and the `--fluid-motion` scalar). The button
reads `--fluid-target-min` for its minimum height, so it never falls below the
24×24 CSS-pixel AA floor.

### AA vs AAA

Truncate ships WCAG 2.2 AA by default and scales to AAA when an ancestor sets
`data-fluid-conformance="aaa"` (the header toggle is live, and the snippets
above switch with it). The toggle reads these tokens:

| Token | AA | AAA | Criterion |
| --- | --- | --- | --- |
| `--fluid-target-min` | 24px | **44px** | 2.5.8 → 2.5.5 |
| `--fluid-focus-ring-width` | 2px | **3px** | 2.4.7 → 2.4.13 |
