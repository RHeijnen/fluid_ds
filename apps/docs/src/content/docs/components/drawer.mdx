`<fluid-drawer>` is a side-anchored panel, like a dialog, but it slides in from
`start` (left), `end` (right), `top`, or `bottom`. Common homes: navigation
menus, settings, filter panels, mobile sub-nav. Built on the native `<dialog>`
element, so the focus trap, Escape-to-close, and backdrop come from the platform.

<Demo>
  <fluid-button onclick="document.querySelector('#hero-drawer').show()">Open filters</fluid-button>
  <fluid-drawer id="hero-drawer" placement="end">
    <span slot="label">Filters</span>
    <p>Filter controls go here.</p>

### Placement

```html
<fluid-drawer placement="start">…</fluid-drawer>   <!-- slides from left -->
<fluid-drawer placement="end">…</fluid-drawer>     <!-- slides from right (default) -->
<fluid-drawer placement="top">…</fluid-drawer>     <!-- slides down -->
<fluid-drawer placement="bottom">…</fluid-drawer>  <!-- slides up -->
```

### Programmatic control

Same surface as `<fluid-dialog>`: `show()` / `hide()` methods, and a
`fluid-show` / `fluid-hide` event pair.

```ts
document.querySelector("fluid-drawer").show();
```

## When to use

- Side-anchored navigation that overlays content (mobile nav drawer).
- Filters / settings panels that need real estate but shouldn't take
  over the screen the way a full modal does.
- A multi-step flow where each step deserves more room than a popover.

Reach for [`<fluid-dialog>`](/components/dialog/) when the content
should feel central and modal. Use [`<fluid-popover>`](/components/popover/)
for small contextual surfaces anchored to a trigger.

## Listening for changes

`<fluid-drawer>` fires **`fluid-show`** / **`fluid-hide`** and exposes `show()` /
`hide()`.

    ```html
    <script type="module">
      const d = document.getElementById("d");
      d.addEventListener("fluid-hide", () => console.log("closed"));
      d.show();
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="Filters.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/drawer";

    export function Filters() {
      const ref = useRef<HTMLElement & { show(): void }>(null);
      useEffect(() => {
        const el = ref.current!;
        const onHide = () => console.log("closed");
        el.addEventListener("fluid-hide", onHide);
        return () => el.removeEventListener("fluid-hide", onHide);
      }, []);
      return <fluid-drawer ref={ref} placement="end"><span slot="label">Filters</span>…</fluid-drawer>;
    }
    ```

## Theming

Every styled property reads a **component-scoped `--fluid-drawer-*` token that
falls back to a main semantic var** (the override ladder), including the
backdrop and separators. `--fluid-drawer-size` sets the width (start/end) or
height (top/bottom). The **complete list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-drawer-bg: var(--fluid-surface-base);
  --fluid-drawer-size: 24rem;
  --fluid-drawer-backdrop: rgb(0 0 0 / 0.5);
}
```

### Beyond tokens: `::part()`

The native element is [`part="base"`](#api); inside are `part="panel"` and
`part="close"`. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

Built on the native `<dialog>` element, so it inherits the platform's modal
semantics. The cross-component contract is in the
[Accessibility guide](/guides/accessibility/).

### Focus & keyboard

- `showModal()` provides the focus trap; focus is restored to the trigger on
  close ([SC 2.4.3](https://www.w3.org/WAI/WCAG22/Understanding/focus-order)).
- <kbd>Escape</kbd> closes the drawer (handled natively); it renders in the
  **top layer**, so it's never clipped.

### Names & roles

- The native `<dialog>` exposes `role="dialog"` + `aria-modal="true"`.
- The `label` slot is the accessible name; with no label, set `aria-label` on
  the host. The built-in close button is labeled "Close drawer".

### Motion & target size

- The slide animation honors `prefers-reduced-motion`: the drawer snaps into
  place instead of transitioning.
- The close button floors its hit area to `--fluid-target-min`: ≥ **24×24**
  (AA) / **44×44** (AAA).

### AA vs AAA

Ships **WCAG 2.2 AA**; an ancestor `[data-fluid-conformance="aaa"]` lifts:

| Token | AA | AAA | Criterion |
| --- | --- | --- | --- |
| `--fluid-target-min` | 24px | **44px** | 2.5.8 → [2.5.5 Target Size (Enhanced)](https://www.w3.org/WAI/WCAG22/Understanding/target-size-enhanced) |
| `--fluid-focus-ring-width` | 2px | **3px** | 2.4.7 → [2.4.13 Focus Appearance](https://www.w3.org/WAI/WCAG22/Understanding/focus-appearance) |
