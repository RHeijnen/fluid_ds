`<fluid-divider>` is the workhorse separator, one pixel of border color,
horizontal by default. It exists as a component (not an inline `<hr>`) so it
carries the right ARIA role, themes from the system tokens, and swaps
orientation by attribute.

<Demo>
  <div style="width: 100%;">
    <p style="margin: 0 0 var(--fluid-space-3);">Section one.</p>
    <fluid-divider></fluid-divider>
    <p style="margin: var(--fluid-space-3) 0 0;">Section two.</p>
  </div>
</Demo>

## When to use

- Visually separating two adjacent sections that don't already have
  their own surface (cards, callouts) doing the visual work.
- Between items in a row when you don't want them to look like buttons
  or pills.

If grouping a list of related content, reach for
[`<fluid-card>`](/components/card/) instead, a contained surface is a
stronger signal than a line.

## Theming

Every styled property reads a **component-scoped `--fluid-divider-*` token that
falls back to a main semantic var** (the override ladder). The **complete list**
is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-divider-color: var(--fluid-border-default);
  --fluid-divider-width: 1px;
}
```

### Beyond tokens: `::part()`

The line is [`part="base"`](#api). Reach it with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS not
covered by a token.

## Accessibility

- The host carries `role="separator"` with `aria-orientation` matching
  the visual orientation.
- Use it for **decorative or sectioning** separation. For semantic
  breaks in a list of nav items, the platform `<hr>` may be more
  appropriate.
