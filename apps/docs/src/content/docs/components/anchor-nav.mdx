`<fluid-anchor-nav>` is an in-page table of contents. It renders a navigation
landmark of anchor links, one per section, then watches the page with an
`IntersectionObserver` and marks the link for the section currently in view with
`aria-current="true"` and an active style. Clicking a link smooth-scrolls to the
section (honoring `prefers-reduced-motion`) and the component emits
**`fluid-active-change`** whenever the active section changes.

  ></fluid-anchor-nav>
</Demo>

## Examples

### Explicit items

Pass an array of `{ id, label, level? }` to the `items` property. Each `id` must
match the `id` of a section in the document. `level` (2 for top-level, 3 for
nested) controls indentation.

<Demo>
  <fluid-anchor-nav
    items='[{"id":"overview","label":"Overview","level":2},{"id":"props","label":"Properties","level":3},{"id":"events","label":"Events","level":3},{"id":"examples","label":"Examples","level":2}]'
  ></fluid-anchor-nav>
</Demo>

```html
<fluid-anchor-nav id="toc"></fluid-anchor-nav>

<script type="module">
  document.getElementById("toc").items = [
    { id: "overview", label: "Overview", level: 2 },
    { id: "props", label: "Properties", level: 3 },
    { id: "examples", label: "Examples", level: 2 }
  ];
</script>
```

### Auto-collect from headings

Leave `items` empty and the component collects every heading matching
`heading-selector` (default `h2,h3`) that has an `id`, inside the `scope`
container (default the document body). Headings without an `id` are skipped.

```html
<article id="content">
  <h2 id="intro">Introduction</h2>
  <h2 id="setup">Setup</h2>
  <h3 id="cli">CLI</h3>
</article>

<fluid-anchor-nav scope="#content" heading-selector="h2,h3"></fluid-anchor-nav>
```

### Offsetting a sticky header

If a sticky header overlaps the top of the viewport, set `offset-top` (in px) so
a section is only counted as "in view" once it clears the header.

```html
<fluid-anchor-nav offset-top="64"></fluid-anchor-nav>
```

## Reacting to the active section

The component fires **`fluid-active-change`** with `{ id }` whenever the section
in view changes (`id` is `null` when nothing tracked is in view).

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-anchor-nav id="toc"></fluid-anchor-nav>

    <script type="module">
      document.getElementById("toc")
        .addEventListener("fluid-active-change", (e) => console.log(e.detail.id));
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="Toc.tsx"
    import { useRef, useEffect, useState } from "react";
    import "@fluid-ds/components/define/anchor-nav";

    export function Toc() {
      const ref = useRef<HTMLElement>(null);
      const [active, setActive] = useState<string | null>(null);
      useEffect(() => {
        const el = ref.current;
        const onChange = (e: Event) =>
          setActive((e as CustomEvent<{ id: string | null }>).detail.id);
        el?.addEventListener("fluid-active-change", onChange);
        return () => el?.removeEventListener("fluid-active-change", onChange);
      }, []);
      return <fluid-anchor-nav ref={ref} />;
    }
    ```
  </TabItem>
  <TabItem label="Vue">
    ```vue title="Toc.vue"
    <script setup lang="ts">
    import { ref } from "vue";
    import "@fluid-ds/components/define/anchor-nav";
    const active = ref<string | null>(null);
    function onChange(e: CustomEvent<{ id: string | null }>) {
      active.value = e.detail.id;
    }
    </script>

    <template>
      <fluid-anchor-nav @fluid-active-change="onChange" />
    </template>
    ```
  </TabItem>
  <TabItem label="Angular">
    ```ts title="toc.component.ts"

    @Component({
      standalone: true,
      schemas: [CUSTOM_ELEMENTS_SCHEMA],

    })
    export class Toc {
      onChange(e: CustomEvent<{ id: string | null }>) { console.log(e.detail.id); }
    ```

## When to use

- A persistent "On this page" sidebar for long-form articles or docs.
- A sticky in-page contents list that highlights the reader's position.

Reach for [`<fluid-tabs>`](/components/tabs/) when the sections are mutually
exclusive panels in the same viewport rather than a single scrolling document.

## Theming

Every styled property reads a **component-scoped `--fluid-anchor-nav-*` token
that falls back to a main semantic var** (the override ladder): idle, hover, and
active link colors, the marker rail, the per-level indent, and the focus ring.
The **complete list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-anchor-nav-fg: var(--fluid-text-secondary);
  --fluid-anchor-nav-active-fg: var(--fluid-accent-base);
  --fluid-anchor-nav-active-marker: var(--fluid-accent-base);
  --fluid-anchor-nav-indent: var(--fluid-space-4);
}
```

### Beyond tokens: `::part()`

The landmark is [`part="base"`](#api) with `part="list"`, `part="item"`, and
`part="link"`. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS not
covered by a token.

## Accessibility

- The list is a `<nav>` landmark with an accessible name (`nav-label`, default
  "On this page"), so it is distinct from other navigation landmarks on the page.
- Entries are real anchor links (`<a href="#id">`), so they work without
  JavaScript and are reachable in the Tab sequence.
- The link for the section in view carries `aria-current="true"`; the active
  style never relies on color alone, the marker rail and weight change too.
- Activating a link smooth-scrolls (honoring `prefers-reduced-motion`, which
  collapses to an instant jump) and moves focus to the target section so
  keyboard and screen-reader users land where sighted users do.
- Focus is shown with a visible ring sized from `--fluid-focus-ring-width`.

### AA vs AAA

- **Target size.** Each link is at least `max(--fluid-space-6, var(--fluid-target-min, 0px))`
  tall, meeting the 24×24 minimum at AA; under `data-fluid-conformance="aaa"`
  the inherited `--fluid-target-min` lifts every link to 44×44.
- **Focus appearance.** The focus ring reads `--fluid-focus-ring-width` /
  `--fluid-focus-ring-offset`, so it thickens automatically in AAA mode without
  per-component branching.
