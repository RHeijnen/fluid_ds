`<fluid-skeleton>` is the placeholder primitive, a tinted rectangle that pulses
(or sweeps) to signal "loading" while you fetch real content. Size it like any
block via `width` / `height`; stack several to mock a card, list row, or
paragraph, then swap in the real markup when the data arrives.

    <fluid-skeleton style="width: 100%;"></fluid-skeleton>

  </div>

## When to use

- Initial page load when the layout shape is known but the data isn't
  back yet.
- Anywhere a spinner would tell the user "loading" without telling
  them what's about to appear.
- For an indeterminate loader without the layout-reservation aspect,
  use [`<fluid-spinner>`](/components/spinner/).
- For known progress, use
  [`<fluid-progress-bar>`](/components/progress-bar/) or
  [`<fluid-progress-ring>`](/components/progress-ring/).

## Theming

Every styled property reads a **component-scoped `--fluid-skeleton-*` token that
falls back to a main semantic var** (the override ladder). Size with standard
`width` / `height`. The **complete list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-skeleton-color: var(--fluid-color-neutral-200);
  --fluid-skeleton-radius: 8px;
  --fluid-skeleton-sheen-color: rgb(255 255 255 / 0.6);
}
```

### Beyond tokens: `::part()`

The placeholder is [`part="base"`](#api). Reach it with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

The cross-component contract is in the
[Accessibility guide](/guides/accessibility/); the notes below are
skeleton-specific.

### Names & roles

- The host is `role="status"` + `aria-busy="true"` + `aria-live="polite"` with a
  default `aria-label` of "Loading", so assistive tech announces "Loading" when
  skeletons appear and falls silent once they're removed
  ([SC 4.1.3](https://www.w3.org/WAI/WCAG22/Understanding/status-messages)).
- Use **one** skeleton (or a single wrapper carrying `aria-busy`) per logical
  loading region, many `role="status"` nodes get noisy.

### Motion

Both the `pulse` and `sheen` effects honor `prefers-reduced-motion` (they fall
back to a static placeholder).
