`<fluid-avatar>` is the identity glyph, a profile photo, a set of initials,
or a generic icon, whichever is available. The fallback chain runs in that
order: try the image, fall back to initials, fall back to whatever's in the
`icon` slot. A second `badge` slot anchors a status dot or count to the
bottom-right corner.

<Demo>
  <fluid-avatar image="https://i.pravatar.cc/80?img=1" label="Ada Lovelace"></fluid-avatar>
  <fluid-avatar label="Grace Hopper"></fluid-avatar>
  <fluid-avatar>
    <fluid-icon slot="icon" name="user"></fluid-icon>
  </fluid-avatar>
</Demo>

### Initials fallback

Pass `label` (full name) and the avatar derives two-letter initials
automatically. Or pass `initials` directly.

  <fluid-avatar initials="AL"></fluid-avatar>


## When to use

- The user's identity (signed-in user, team members, comment authors).
- Anywhere a "who" is part of the row: assignees, mentions, sharing
  lists.

If you need a plain status indicator without a user attached, reach for
[`<fluid-badge>`](/components/badge/) directly.

## Theming

Every styled property reads a **component-scoped `--fluid-avatar-*` token that
falls back to a main semantic var** (the override ladder). The **complete
list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-avatar-bg: var(--fluid-surface-muted);
  --fluid-avatar-fg: var(--fluid-text-primary);
  --fluid-avatar-border: var(--fluid-border-default);
}
```

### Beyond tokens: `::part()`

The avatar exposes [`part="base"`](#api) plus `part="image"` and
`part="initials"`. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token, e.g. a ring around the image.

## Accessibility

- The host carries `role="img"` with an `aria-label` derived from `label`
  (falling back to the initials if no label).
- The `<img>` itself has `alt=""` because the host already provides the
  accessible name, this prevents screen readers from announcing the
  image twice.
- Set `label` whenever you can. Without it, the accessible name is just
  the initials, which isn't very helpful on its own.
