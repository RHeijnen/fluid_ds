`<fluid-avatar-group>` clusters slotted [`<fluid-avatar>`](/components/avatar/)
elements into one overlapping stack. Set `max` to cap how many show: the first N
stay visible and the rest collapse into a trailing **"+N"** counter circle. The
whole stack is one `role="group"` with an "N members" name, so assistive tech
announces it as a unit.

<Demo>
  <fluid-avatar-group max="3">
    <fluid-avatar label="Ada Lovelace"></fluid-avatar>
    <fluid-avatar label="Grace Hopper"></fluid-avatar>
    <fluid-avatar label="Alan Turing"></fluid-avatar>
    <fluid-avatar label="Katherine Johnson"></fluid-avatar>
    <fluid-avatar label="Edsger Dijkstra"></fluid-avatar>
  </fluid-avatar-group>
</Demo>

### Sizes

`size` scales the overflow circle and forwards to any avatar that hasn't set its
own size. The five sizes match [`<fluid-avatar>`](/components/avatar/): `xs`,
`sm`, `md`, `lg`, `xl`.

## When to use

- A row of collaborators, reviewers, or attendees where space is tight.
- A "who's here" cluster on a card, list row, or document header.
- Any set of people that reads better as a compact stack than a wrapping list.

Use a single [`<fluid-avatar>`](/components/avatar/) when there's only one
person, and a plain list when every name needs to be individually readable or
actionable.

## Theming

Every styled property reads a **component-scoped `--fluid-avatar-group-*` token
that falls back to a main semantic var** (the override ladder): the ring drawn
between overlapping avatars, the overlap distance, and the overflow circle's
colors. The **complete list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-avatar-group-overlap: 0.75rem;
  --fluid-avatar-group-ring: var(--fluid-surface-base);
  --fluid-avatar-group-overflow-bg: var(--fluid-surface-muted);
  --fluid-avatar-group-overflow-fg: var(--fluid-text-primary);
}
```

### Beyond tokens: `::part()`

The container is [`part="base"`](#api) with `part="overflow"` for the "+N"
circle. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

- The group is a single `role="group"` with an accessible name (`label`, or an
  auto-derived "N members") so assistive tech announces the cluster as a whole
  instead of reading every avatar in turn.
- Avatars beyond `max`, and the "+N" overflow circle, are marked
  `aria-hidden="true"`: the visible count is decorative, and the group name
  already conveys the total.
- Each visible [`<fluid-avatar>`](/components/avatar/) keeps its own
  `role="img"` accessible name from its `label`.
