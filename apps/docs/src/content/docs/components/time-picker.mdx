`<fluid-time-picker>` is a form-associated text field that opens a
`role="listbox"` of time options in a popover. You can type to filter or pick
from the list; either way the form value is always a canonical 24-hour
`"HH:MM"` string. The `format` attribute controls only the *visible* text
("12h" shows `1:30 PM`, "24h" shows `13:30`), never the value, so presentation
never corrupts what you store.

<Demo>
  <fluid-time-picker value="09:30" aria-label="Time"></fluid-time-picker>
</Demo>

the list; <kbd>↓</kbd> in the field opens it too. Type to filter the options.

## Listening for changes

The picker fires **`fluid-change`** with `{ value }` in `detail` when the
committed time changes (a list pick, or a valid typed entry on <kbd>Enter</kbd>
/ blur). `value` is the canonical 24h `"HH:MM"` string (or `null` when cleared).
It also fires `fluid-open` / `fluid-close` around the popover.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-time-picker id="start" aria-label="Start time"></fluid-time-picker>

    <script type="module">
      document.getElementById("start").addEventListener("fluid-change", (e) => {
        console.log(e.detail.value);
      });
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="StartTime.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/time-picker";

    export function StartTime() {
      const ref = useRef<HTMLElement>(null);
      useEffect(() => {
        const el = ref.current!;
        const onChange = (e: Event) =>
          console.log((e as CustomEvent).detail.value);
        el.addEventListener("fluid-change", onChange);
        return () => el.removeEventListener("fluid-change", onChange);
      }, []);
      return <fluid-time-picker ref={ref} aria-label="Start time" />;
    }
    ```
  </TabItem>
  <TabItem label="Vue">

## Inside a form

The picker is form-associated: it submits `name=<24h value>`, and `required`
blocks submission until a time is chosen. The submitted value is always
canonical 24h `HH:MM`, independent of the `format` shown.

<Demo>
  <form
    style="display:flex; gap:0.5rem; align-items:center; flex-wrap:wrap;"
    onsubmit="event.preventDefault(); this.querySelector('output').textContent = 'start=' + (new FormData(this).get('start') ?? '(none)');"
  >
    <fluid-time-picker name="start" required placeholder="Start time" aria-label="Start time"></fluid-time-picker>
    <fluid-button type="submit">Save</fluid-button>
    <output style="font-size:0.875rem; color: var(--fluid-text-secondary);"></output>
  </form>
</Demo>

```html
<form>
  <fluid-time-picker name="start" required></fluid-time-picker>
  <fluid-button type="submit">Save</fluid-button>
</form>
```

wiring across frameworks.

## Theming

Every styled property of the picker reads a **component-scoped
`--fluid-time-picker-*` token that falls back to a main semantic var** (the
override ladder); most fall through to the shared `--fluid-input-*` /
`--fluid-field-*` tokens, so the field matches your inputs by default and only
the listbox surface and options are picker-specific. The **complete,
authoritative list** lives in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-time-picker-border-focus: #db2777;
  --fluid-time-picker-option-selected-bg: #db2777;
  --fluid-time-picker-listbox-bg: var(--fluid-surface-base);
}
```

<Aside type="tip">
  The [Theme builder](/playground/) lets you tweak the field, listbox, and
  option tokens visually and exports the CSS above.
</Aside>

### Beyond tokens: `::part()`

The field is [`part="base"`](#api), with `input`, `trigger`, `listbox`, and
`option` sub-parts. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token:

```css
/* Widen the popover. */
fluid-time-picker::part(listbox) {
  min-width: 12rem;
}
```

## When to use

- A **single time of day** in a form: an appointment, a start time, a reminder.
- Anywhere you need both **typed entry and a pickable list** with one
  accessible control that submits a clean 24h value.

### When *not* to use

- A **calendar date**: use [`<fluid-date-picker>`](/components/date-picker/).
- **Free text** that isn't a time: use [`<fluid-input>`](/components/input/).

## Accessibility

`<fluid-time-picker>` follows the WAI-ARIA APG **Combobox** pattern with a
`role="listbox"` popup. The field stays a text input (`role="combobox"`,
`aria-haspopup="listbox"`, `aria-expanded`, `aria-controls`); DOM focus stays on
the input and the active option is tracked with `aria-activedescendant`. The
notes below are picker-specific; the cross-component contract is in the
[Accessibility guide](/guides/accessibility/).

### Keyboard

| Key | Action |
| --- | --- |
| _type a time_ | Filter the list; <kbd>Enter</kbd> commits the active option (or a typed match). |
| <kbd>↓</kbd> / <kbd>↑</kbd> | Open the list and move the active option. |
| <kbd>Home</kbd> / <kbd>End</kbd> | Jump to the first / last option. |
| <kbd>Enter</kbd> | Commit the active option and close. |
| <kbd>Escape</kbd> | Close the popover without changing the value. |

The field's focus ring is `:focus-within` and reads
`--fluid-time-picker-focus-ring-width` (falling back to
`--fluid-focus-ring-width`).

### Names & roles

The field needs an accessible name, a visible label wired with
`aria-labelledby`, or an `aria-label`
([SC 4.1.2](https://www.w3.org/WAI/WCAG22/Understanding/name-role-value)). The
trigger button is named "Choose time", the listbox is labeled, and each option
carries `aria-selected` to mark the current value.

### State semantics

`required` participates in form validation (a missing value reports
`valueMissing`). `disabled` removes the field from the tab order and blocks the
popover. `aria-expanded` on both the field and trigger reflects the open state.

### Motion & target size

- The popover open/close transition honors `prefers-reduced-motion: reduce`.
- The trigger and every option is at least **24×24** CSS px by default
  ([SC 2.5.8 Target Size (Minimum)](https://www.w3.org/WAI/WCAG22/Understanding/target-size-minimum.html),
  AA), reading `--fluid-target-min` as a floor.

### AA vs AAA

Ships **WCAG 2.2 AA**; flip the **Conformance** toggle to **AAA** and the
trigger target grows, the option targets grow, and the focus ring thickens. The
toggle sets the real `data-fluid-conformance` attribute on `<html>`:

| Token | AA | AAA | Criterion |
| --- | --- | --- | --- |
| `--fluid-target-min` | 24px | **44px** | 2.5.8 → [2.5.5 Target Size (Enhanced)](https://www.w3.org/WAI/WCAG22/Understanding/target-size-enhanced) |
| `--fluid-focus-ring-width` | 2px | **3px** | 2.4.7 → [2.4.13 Focus Appearance](https://www.w3.org/WAI/WCAG22/Understanding/focus-appearance) |
