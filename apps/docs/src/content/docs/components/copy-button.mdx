`<fluid-copy-button>` is the focused, single-purpose copy-to-clipboard
button. It resolves the text to copy in this order: the `value` attribute,
the element referenced by `from` (an ID), then its slotted text content.
On success it flips to a check icon for ~1.5 seconds; on failure it stays
silent.

For copy-as-affordance on a code snippet, reach for
[`<fluid-code-block>`](/components/code-block/), it embeds one of these
already.

<Demo>
  <fluid-copy-button value="hello@example.com" aria-label="Copy email"></fluid-copy-button>
  <fluid-copy-button value="hello@example.com">
    <fluid-icon name="copy"></fluid-icon>
    Copy email
  </fluid-copy-button>
</Demo>

### Slot-based label

The default slot becomes the idle label, use it to add a visible "Copy"
text alongside the icon, or to replace the icon entirely.

<Demo>
  <fluid-copy-button value="hello@example.com">
    <fluid-icon name="copy"></fluid-icon>
    Copy email
  </fluid-copy-button>
</Demo>

You can also override the `success` and `error` slots if you want bespoke
labels for those states:

```html
<fluid-copy-button value="…">
  <span>Copy link</span>
  <span slot="success">Copied!</span>
</fluid-copy-button>
```

### Custom feedback duration

`feedback-duration` is in milliseconds; default 1500.

```html
<fluid-copy-button value="…" feedback-duration="3000">…</fluid-copy-button>
```

## Listening for copy

The button fires **`fluid-copy`** after every attempt, with `event.detail.success`
(did the clipboard write succeed?) and `event.detail.text` (what was copied).

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-copy-button id="btn" value="hello@example.com" aria-label="Copy email"></fluid-copy-button>

    <script type="module">
      document.getElementById("btn").addEventListener("fluid-copy", (e) => {
        console.log("success:", e.detail.success, "text:", e.detail.text);
      });
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="CopyEmail.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/copy-button";

    export function CopyEmail() {
      const ref = useRef<HTMLElement>(null);
      useEffect(() => {
        const el = ref.current!;
        const onCopy = (e: any) => console.log(e.detail.success);
        el.addEventListener("fluid-copy", onCopy);
        return () => el.removeEventListener("fluid-copy", onCopy);
      }, []);
      return <fluid-copy-button ref={ref} value="hello@example.com" aria-label="Copy email" />;
    }
    ```
  </TabItem>
  <TabItem label="Vue">
    ```vue title="CopyEmail.vue"

    </script>

    <template>
      <fluid-copy-button value="hello@example.com" aria-label="Copy email" @fluid-copy="onCopy" />
    </template>
    ```
  </TabItem>
  <TabItem label="Angular">

## When to use

- Inline copy of a single value: a token, an address, an API key.
- For code-block copy, use [`<fluid-code-block>`](/components/code-block/).

## Theming

Every styled property reads a **component-scoped `--fluid-copy-button-*` token
that falls back to a main semantic var** (the override ladder); the success
flash uses the **theme-independent** success tone. The **complete list** is in
the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-copy-button-fg: var(--fluid-text-primary);
  --fluid-copy-button-hover-bg: var(--fluid-surface-muted);
  --fluid-copy-button-success-color: var(--fluid-success-base);
}
```

### Beyond tokens: `::part()`

The button is [`part="base"`](#api). Reach it with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

- The internal button has an `aria-label` of "Copy to clipboard" that
  flips to "Copied" for the feedback duration.
- Set your own `aria-label` on the host when copying something specific
  ("Copy email", "Copy API key") for better screen-reader context.
- The copy action calls `navigator.clipboard.writeText` and fires
  `fluid-copy` with `success: false` if it fails, no exception is
  thrown.

### AA vs AAA

Ships **WCAG 2.2 AA**; an ancestor `[data-fluid-conformance="aaa"]` lifts the
focus indicator:

| Token | AA | AAA | Criterion |
| --- | --- | --- | --- |
| `--fluid-focus-ring-width` | 2px | **3px** | 2.4.7 → [2.4.13 Focus Appearance](https://www.w3.org/WAI/WCAG22/Understanding/focus-appearance) |
