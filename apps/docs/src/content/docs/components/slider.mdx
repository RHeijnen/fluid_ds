`<fluid-slider>` wraps a native `<input type="range">` so you get every platform
behavior for free, keyboard, drag, touch, then styles it as a themable track
with a coloured fill behind a capsule grab-handle thumb. Form-associated.

<Demo>
  <fluid-slider value="40" show-value aria-label="Volume" style="width: 100%;"></fluid-slider>
</Demo>

### Custom formatter

Pass a function as the `valueFormatter` property (not an attribute). It drives
both the visible label and `aria-valuetext`.

```ts
const slider = document.querySelector("fluid-slider");
slider.showValue = true;
slider.valueFormatter = (v) => `${v}%`;
```

## Listening for changes

Slider fires **`fluid-input`** on every value change (drag, keypress) and
**`fluid-change`** when the change is committed (release / blur). Both carry
`{ value }` in `detail`.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-slider id="vol" aria-label="Volume"></fluid-slider>

    <script type="module">
      const el = document.getElementById("vol");
      el.addEventListener("fluid-input", (e) => console.log("dragging:", e.detail.value));
      el.addEventListener("fluid-change", (e) => console.log("released:", e.detail.value));
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="VolumeSlider.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/slider";

    export function VolumeSlider() {
      const ref = useRef<HTMLElement>(null);
      useEffect(() => {
        const el = ref.current!;
        const onInput = (e: Event) =>
          console.log((e as CustomEvent).detail.value);
        el.addEventListener("fluid-input", onInput);
        return () => el.removeEventListener("fluid-input", onInput);
      }, []);
      return <fluid-slider ref={ref} aria-label="Volume" />;
    }
    ```
  </TabItem>
  <TabItem label="Vue">
    ```vue title="VolumeSlider.vue"
    <script setup lang="ts">
    import "@fluid-ds/components/define/slider";
    const onInput = (e: CustomEvent) => console.log(e.detail.value);
    </script>

    <template>
      <fluid-slider aria-label="Volume" @fluid-input="onInput" />
    </template>
    ```
  </TabItem>
  <TabItem label="Angular">
    ```ts title="volume-slider.component.ts"
    import { Component, CUSTOM_ELEMENTS_SCHEMA } from "@angular/core";
    import "@fluid-ds/components/define/slider";

    @Component({
      standalone: true,
      schemas: [CUSTOM_ELEMENTS_SCHEMA],
      selector: "volume-slider",

    })

      onInput(e: CustomEvent) {
        console.log(e.detail.value);
      }
    }
    ```

## Inside a form

The slider is form-associated: it submits `name=<value>`.

```html
<form>
  <fluid-slider name="volume" value="40"></fluid-slider>
  <fluid-button type="submit">Save</fluid-button>
</form>
```

## Theming

Every styled property of the slider reads a **component-scoped `--fluid-slider-*`
token that falls back to a main semantic var** (the override ladder). The
**complete list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-slider-track-color: var(--fluid-color-neutral-200);
  --fluid-slider-fill-color: var(--fluid-accent-base);
  --fluid-slider-thumb-color: var(--fluid-accent-base);
}
```

<Aside type="tip">
  Select a `fluid-slider` in the [Theme builder](/playground/) to tweak the
  track, fill, thumb, and focus tokens visually.
</Aside>

### Beyond tokens: `::part()`

The container is [`part="base"`](#api) and the range control is `part="input"`.
Reach them with [`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part)
for CSS not covered by a token, including the vendor `::-webkit-slider-thumb` /
`::-moz-range-thumb` pseudo-elements through the part.

## When to use

- A numeric value where **roughly** the right number matters more than the exact
  number (brightness, opacity, zoom level).

### When *not* to use

- **Exact numbers users want to type**: use
  [`<fluid-number-input>`](/components/number-input/).
- **5-star or 10-point scoring**: use [`<fluid-rating>`](/components/rating/).

## Accessibility

The underlying control is a real `<input type="range">`, so it inherits full
native semantics, announced as a slider with min, max, and current value. The
notes below are slider-specific; the cross-component contract is in the
[Accessibility guide](/guides/accessibility/).

### Keyboard

| Key | Action |
| --- | --- |
| <kbd>←</kbd> / <kbd>↓</kbd> | Decrease by `step`. |
| <kbd>→</kbd> / <kbd>↑</kbd> | Increase by `step`. |
| <kbd>Shift</kbd> + arrow | Step by ×10. |
| <kbd>Home</kbd> / <kbd>End</kbd> | Jump to min / max. |
| <kbd>PageUp</kbd> / <kbd>PageDown</kbd> | Step by ~10%. |

### Names & roles

Always set `aria-label` (or pair with a `<label for="…">`). The `valueFormatter`,
when set, drives `aria-valuetext` for richer announcements like "50%" instead of
just "50" ([SC 4.1.2](https://www.w3.org/WAI/WCAG22/Understanding/name-role-value)).

### Motion & target size

- Thumb grow/shrink transitions honor `prefers-reduced-motion`.
- The track is thin, so the full input row carries the target floor:
  `--fluid-target-min` makes the pointer/drag area ≥ **24px** (AA) / **44px**
  (AAA) tall without thickening the visible track.

### AA vs AAA

Ships **WCAG 2.2 AA**; flip the **Conformance** toggle to **AAA**:

| Token | AA | AAA | Criterion |
| --- | --- | --- | --- |
| `--fluid-target-min` | 24px | **44px** | 2.5.8 → [2.5.5 Target Size (Enhanced)](https://www.w3.org/WAI/WCAG22/Understanding/target-size-enhanced) |
| `--fluid-focus-ring-width` | 2px | **3px** | 2.4.7 → [2.4.13 Focus Appearance](https://www.w3.org/WAI/WCAG22/Understanding/focus-appearance) |
