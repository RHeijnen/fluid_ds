`<fluid-radio>` is one option; `<fluid-radio-group>` is the container that holds
the set and manages selection, keyboard navigation, and form participation.
Always wrap radios in a group, a radio on its own has no concept of its set.
The group implements the WAI-ARIA radiogroup pattern (roving tabindex, arrow-key
nav) and is form-associated.

<Demo>
  <fluid-radio-group value="medium" aria-label="Size">
    <fluid-radio value="small">Small</fluid-radio>
    <fluid-radio value="medium">Medium</fluid-radio>
    <fluid-radio value="large">Large</fluid-radio>
  </fluid-radio-group>
</Demo>

## Listening for changes

The group fires **`fluid-change`** with `{ value }` in `detail` when the
selection changes.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-radio-group id="plan" aria-label="Plan">…</fluid-radio-group>

    <script type="module">
      document.getElementById("plan").addEventListener("fluid-change", (e) => {
        console.log("selected:", e.detail.value);
      });
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="PlanPicker.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/radio";

    export function PlanPicker() {
      const ref = useRef<HTMLElement>(null);
      useEffect(() => {
        const el = ref.current!;
        const onChange = (e: Event) =>
          console.log((e as CustomEvent).detail.value);
        el.addEventListener("fluid-change", onChange);
        return () => el.removeEventListener("fluid-change", onChange);
      }, []);
      return (
        <fluid-radio-group ref={ref} aria-label="Plan">
          <fluid-radio value="free">Free</fluid-radio>
          <fluid-radio value="pro">Pro</fluid-radio>
        </fluid-radio-group>
      );
    }
    ```
  </TabItem>
  <TabItem label="Vue">

## Inside a form

The **group** is form-associated: it submits `name=<selected value>`, and
`required` blocks submission until a choice is made.

<Demo>
  <form
    style="display:flex; flex-direction:column; gap:0.75rem; align-items:flex-start;"
    onsubmit="event.preventDefault(); this.querySelector('output').textContent = 'plan=' + (new FormData(this).get('plan') ?? '(none)');"
  >
    <fluid-radio-group name="plan" required>
      <span slot="label">Pick a plan</span>
      <fluid-radio value="free">Free</fluid-radio>
      <fluid-radio value="pro">Pro</fluid-radio>
      <fluid-radio value="enterprise">Enterprise</fluid-radio>
    </fluid-radio-group>
    <div>
      <fluid-button type="submit">Continue</fluid-button>
      <output style="font-size:0.875rem; color: var(--fluid-text-secondary); margin-left:0.5rem;"></output>
    </div>
  </form>
</Demo>

## Theming

Every styled property of the radio reads a **component-scoped `--fluid-radio-*`
token that falls back to a main semantic var** (the override ladder); the
checked color is the brand accent. The group exposes
`--fluid-radio-group-label-fg` for its legend. The **complete list** is in the
[API tables below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-radio-accent: var(--fluid-accent-base);
}
```

<Aside type="tip">
  Select a radio in the [Theme builder](/playground/) to tweak the circle, dot,
  and focus tokens visually.
</Aside>

### Beyond tokens: `::part()`

A radio's label is [`part="base"`](#api) and its circle is `part="control"`.
Reach them with [`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part)
for CSS not covered by a token.

## When to use

- **One-of-many** selection where all options should be visible at once.

### When *not* to use

- A **long list** (> ~7): use [`<fluid-select>`](/components/select/) (denser)
  or [`<fluid-typeahead>`](/components/typeahead/) (filterable).
- **Boolean** on/off: use [`<fluid-checkbox>`](/components/checkbox/) or
  [`<fluid-switch>`](/components/switch/).

## Accessibility

The group is `role="radiogroup"` and each option is `role="radio"`, following
the WAI-ARIA APG. The notes below are radio-specific; the cross-component
contract is in the [Accessibility guide](/guides/accessibility/).

### Keyboard

| Key | Action |
| --- | --- |
| <kbd>Tab</kbd> | Enter/leave the group (roving tabindex, only the selected option is a tab stop). |
| <kbd>↑</kbd> / <kbd>←</kbd> | Select + focus the previous option. |
| <kbd>↓</kbd> / <kbd>→</kbd> | Select + focus the next option. |
| <kbd>Home</kbd> / <kbd>End</kbd> | Select the first / last option. |
| <kbd>Space</kbd> | Select the focused option. |

The focus ring renders around the circle and reads `--fluid-focus-ring-width`.

### Names

The group needs an accessible name: a `label` slot (wired via
`aria-labelledby`) or an `aria-label`. Each radio's name comes from its slotted
text ([SC 4.1.2](https://www.w3.org/WAI/WCAG22/Understanding/name-role-value)).

### State semantics

`aria-checked` reflects each option; the group manages `required` validity and
submits the selected value.

### Motion & target size

- The dot-scale transition honors `prefers-reduced-motion`.
- The circle is small (~18px), so each option's clickable label carries the
  target floor: ≥ **24×24** (AA) / **44×44** (AAA) via `--fluid-target-min`.

### AA vs AAA

Ships **WCAG 2.2 AA**; flip the **Conformance** toggle to **AAA**:

| Token | AA | AAA | Criterion |
| --- | --- | --- | --- |
| `--fluid-target-min` | 24px | **44px** | 2.5.8 → [2.5.5 Target Size (Enhanced)](https://www.w3.org/WAI/WCAG22/Understanding/target-size-enhanced) |
| `--fluid-focus-ring-width` | 2px | **3px** | 2.4.7 → [2.4.13 Focus Appearance](https://www.w3.org/WAI/WCAG22/Understanding/focus-appearance) |
