`<fluid-scroller>` is a scrollable container that adds fade-out shadow
indicators at the edges where more content exists. Useful for
horizontal tab strips, wide tables, vertical lists, anywhere you want
to hint "scroll for more" without baking arrows into the design.

## When to use

- A horizontal strip of tags, tabs, or filters that doesn't fit on
  narrow viewports.
- A vertical list where the user might miss that more is below.

For an auto-rotating slideshow, reach for
[`<fluid-carousel>`](/components/carousel/) instead, it provides
navigation buttons and snap behavior.

## Theming

The fade overlays read **component-scoped `--fluid-scroller-*` tokens** (the
override ladder): set `--fluid-scroller-fade-color` to match the surface the
scroller sits on so the fade blends seamlessly. The **complete list** is in the
[API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-scroller-fade-color: var(--fluid-surface-base);
  --fluid-scroller-fade-size: 1.5rem;
}
```

### Beyond tokens: `::part()`

The scroll container is [`part="base"`](#api). Reach it with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token, e.g. a custom scroll-snap behavior.

## Accessibility

- The container is a plain `<div>` with `overflow: auto`: it's
  keyboard-scrollable (<kbd>↑</kbd>/<kbd>↓</kbd> for vertical,
  <kbd>←</kbd>/<kbd>→</kbd> for horizontal) when focused.
- The fade overlays are `pointer-events: none` and have no semantics:
  they're purely visual.
