`<fluid-intersection-observer>` wraps the platform's `IntersectionObserver` so
you can use it without imperative wiring. Drop elements inside and you'll get a
**`fluid-intersect`** event whenever any of them crosses the viewport (or your
configured root) at the configured thresholds, good for lazy-loading, scroll-
spy, infinite scroll, and animate-on-enter.

</Aside>

## Examples

### Reveal on scroll

```html
<fluid-intersection-observer threshold="0.25">
  <section data-reveal>…</section>
</fluid-intersection-observer>

<script type="module">
  document
    .querySelector("fluid-intersection-observer")
    .addEventListener("fluid-intersect", (e) => {
      for (const entry of e.detail.entries) {
        if (entry.isIntersecting) entry.target.dataset.visible = "true";
      }
    });
</script>
```

### Lazy-load images

```html
<fluid-intersection-observer root-margin="200px">
  <img data-src="/hero.jpg" alt="…">
</fluid-intersection-observer>

<script type="module">
  document
    .querySelector("fluid-intersection-observer")
    .addEventListener("fluid-intersect", (e) => {
      for (const entry of e.detail.entries) {
        if (entry.isIntersecting) {
          entry.target.src = entry.target.dataset.src;
        }
      }
    });
</script>
```

### Multiple thresholds

Pass a comma-separated list to fire at multiple progress points.

```html
<fluid-intersection-observer threshold="0,0.25,0.5,0.75,1">…</fluid-intersection-observer>
```

### Pause observation

```html
<fluid-intersection-observer disabled>…</fluid-intersection-observer>
```

## When to use

- Lazy-load images / iframes / heavy widgets.
- Highlight the section the user is reading.
- Infinite scroll: watch a sentinel at the bottom of the list.

For "is the element being resized?" use
[`<fluid-resize-observer>`](/components/resize-observer/). For "did
something change in the DOM?" use
[`<fluid-mutation-observer>`](/components/mutation-observer/).

## Theming

This is a behavior component, not a visual one, it has no tokens.

## Accessibility

- The host renders with `display: contents`, so it doesn't affect
  layout or accessibility tree shape, its children are observed
  directly.
- No ARIA role of its own. Whatever you slot in keeps its semantics.
