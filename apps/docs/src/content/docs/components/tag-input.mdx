`<fluid-tag-input>` is a token entry field. You type a label and commit it with
**Enter** or a **comma**; each committed token renders as a removable
[`<fluid-tag>`](/components/tag/) chip ahead of the text field. **Backspace** on
an empty field removes the last token, and clicking a chip's × removes it. It is
form-associated: its submitted value is the comma-joined string, while the
`value` property is the array of tokens. It emits **`fluid-change`** with the
array on every add or remove.

## Examples

### Prefilled tokens

The `value` attribute accepts a comma-separated string, which is split into the
token array. The `value` property is always the array.

<Demo>
  <fluid-tag-input aria-label="Skills" value="design,systems,web components,lit"></fluid-tag-input>
</Demo>

```html
<fluid-tag-input aria-label="Skills" value="design,systems,lit"></fluid-tag-input>
```

### Capped count

Set `max` to cap the number of tokens. Once the cap is reached the text field is
disabled, so no further tokens can be added.

<Demo>
  <fluid-tag-input aria-label="Up to three" value="one,two" max="3" placeholder="Up to 3 tags"></fluid-tag-input>
</Demo>

```html
<fluid-tag-input aria-label="Up to three" max="3" value="one,two"></fluid-tag-input>
```

### Allow duplicates

By default a token already present is rejected. Set `allow-duplicates` to permit
repeats.

<Demo>
  <fluid-tag-input aria-label="Duplicates allowed" value="react,react" allow-duplicates></fluid-tag-input>
</Demo>

```html
<fluid-tag-input aria-label="Duplicates allowed" allow-duplicates></fluid-tag-input>
```

## Listening for change

`<fluid-tag-input>` fires **`fluid-change`** whenever the token set changes (add
or remove). `event.detail.value` is the current array of tokens.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-tag-input id="t" aria-label="Tags"></fluid-tag-input>

    <script type="module">
      document.getElementById("t").addEventListener("fluid-change", (e) => {
        console.log(e.detail.value); // ["react", "lit", …]
      });
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="TagField.tsx"
    import { useRef, useEffect, useState } from "react";
    import "@fluid-ds/components/define/tag-input";

    export function TagField() {
      const ref = useRef<HTMLElement>(null);
      const [tags, setTags] = useState<string[]>(["react", "lit"]);
      useEffect(() => {
        const el = ref.current!;
        const onChange = (e: any) => setTags(e.detail.value);
        el.addEventListener("fluid-change", onChange);
        return () => el.removeEventListener("fluid-change", onChange);
      }, []);
      return <fluid-tag-input ref={ref} aria-label="Tags" value={tags.join(",")} />;
    }
    ```
  </TabItem>
  <TabItem label="Vue">
    ```vue title="TagField.vue"
    <script setup lang="ts">
    import { ref } from "vue";
    import "@fluid-ds/components/define/tag-input";
    const tags = ref<string[]>(["react", "lit"]);
    </script>

    <template>
      <fluid-tag-input
        aria-label="Tags"
        :value="tags.join(',')"
        @fluid-change="tags = $event.detail.value"
      />
    </template>
    ```
  </TabItem>
  <TabItem label="Angular">

## Forms

`<fluid-tag-input>` is form-associated. Give it a `name` and it submits with the
form: the submitted value is the **comma-joined string** of tokens (so the
server receives `tags=react,lit`). It resets to its initial `value` attribute on
form reset.

```html
<form>
  <fluid-tag-input name="tags" value="react,lit" aria-label="Tags"></fluid-tag-input>
  <button type="submit">Save</button>
</form>
```

## When to use

- Recipient pills in an email-to field.
- Free-form keyword / skill / category entry.
- Filter chips the user types rather than picks from a fixed set.

Reach for [`<fluid-select>`](/components/select/) when the choices come from a
fixed list rather than free text, and a single [`<fluid-tag>`](/components/tag/)
when you only need to display chips, not capture them.

## Theming

Every styled property reads a **component-scoped `--fluid-tag-input-*` token
that falls back to a main semantic var** (the override ladder). The chips are
themed through the [`--fluid-tag-*` tokens](/components/tag/#api). The
**complete list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-tag-input-bg: var(--fluid-surface-base);
  --fluid-tag-input-border-focus: var(--fluid-accent-base);
}
```

### Beyond tokens: `::part()`

The field shell is [`part="base"`](#api), with `part="tags"` for the chip
wrapper, `part="tag"` for each chip, and `part="input"` for the text field.
Reach them with [`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part)
for CSS not covered by a token.

## Accessibility

- The field shell is a `role="group"`; give it an accessible name with
  `aria-label` (or an adjacent visible label), which also names the inner text
  field.
- The text `<input>` is the labelable control that owns keyboard entry.
- **Keyboard:** type a label and press **Enter** or **comma** to commit it;
  **Backspace** on an empty field removes the last token. Each chip's remove
  button is a real `<button>` reachable with **Tab**.
- After a removal, focus returns to the text field so keyboard users are never
  stranded.
- Motion (the focus-ring transition) honors
  `prefers-reduced-motion: reduce` and the `--fluid-motion` scalar.

### AA vs AAA

The control reads the conformance tokens rather than hard-coding sizes: the
field height floors on `--fluid-target-min` and the focus ring uses
`--fluid-focus-ring-width`. An ancestor with `data-fluid-conformance="aaa"`
lifts the field to a 44×44 target with an enhanced focus ring, with no
per-component change.
