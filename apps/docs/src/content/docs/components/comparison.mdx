`<fluid-comparison>` is the classic before/after slider, two images
stacked, with a draggable divider that wipes from one to the other. Drop
your two images (or any equally-sized content) into the `before` and
`after` slots and the component handles the dragging, clipping, click-to-
jump, and keyboard control.

<Demo>
  <fluid-comparison style="max-width: 32rem;">
    <img slot="before" src="https://placehold.co/600x400/1f2937/f3f4f6?text=Before" alt="Before" />
    <img slot="after" src="https://placehold.co/600x400/0ea5e9/f3f4f6?text=After" alt="After" />
  </fluid-comparison>
</Demo>

### Initial position

`position` is a percentage from the start side (0–100). The default is
50.

### Custom handle

Override the default chevron icon by slotting your own content into
`handle`.

```html
<fluid-comparison>
  <img slot="before" src="…" alt="…">
  <img slot="after"  src="…" alt="…">
  <fluid-icon slot="handle" name="arrow-left-right"></fluid-icon>
</fluid-comparison>
```

## Listening for drag

The slider fires **`fluid-position-change`** as the handle moves (drag or
keyboard), with the new percentage on `event.detail.position`.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-comparison id="cmp">
      <img slot="before" src="/before.jpg" alt="…">
      <img slot="after"  src="/after.jpg"  alt="…">
    </fluid-comparison>

    <script type="module">
      document.getElementById("cmp").addEventListener("fluid-position-change", (e) => {
        console.log("now at", e.detail.position, "%");
      });
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="Compare.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/comparison";

    export function Compare() {
      const ref = useRef<HTMLElement>(null);
      useEffect(() => {
        const el = ref.current!;
        const onMove = (e: any) => console.log(e.detail.position);
        el.addEventListener("fluid-position-change", onMove);
        return () => el.removeEventListener("fluid-position-change", onMove);
      }, []);
      return (
        <fluid-comparison ref={ref}>
          <img slot="before" src="/before.jpg" alt="…" />
          <img slot="after" src="/after.jpg" alt="…" />
        </fluid-comparison>
      );
    }
    ```
  </TabItem>

## When to use

- Showing two states of the same content (light/dark, retouched/raw,
  before/after edit).
- Compose two images or videos with the same aspect ratio. Mismatched
  sizes will look off, pre-process to match.

For an auto-rotating carousel of separate items, reach for
[`<fluid-carousel>`](/components/carousel/) instead.

## Theming

The divider and handle read **component-scoped `--fluid-comparison-*` tokens**
(the override ladder); the divider and handle foreground ride the accent track
by default. The **complete list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-comparison-divider-color: var(--fluid-accent-base);
  --fluid-comparison-handle-bg: var(--fluid-surface-base);
  --fluid-comparison-handle-fg: var(--fluid-accent-base);
}
```

### Beyond tokens: `::part()`

The slider exposes [`part="base"`](#api) plus `part="before"`, `part="after"`,
`part="divider"`, and `part="handle"`. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

- The handle carries `role="slider"` with `aria-valuemin`,
  `aria-valuemax`, and `aria-valuenow` reflecting the percentage.
- Keyboard, when the handle is focused:
  - <kbd>←</kbd>/<kbd>→</kbd> move 1%.
  - <kbd>Shift</kbd>+<kbd>←</kbd>/<kbd>→</kbd> move 10%.
  - <kbd>Home</kbd>/<kbd>End</kbd> snap to 0% / 100%.
- Always set meaningful `alt` text on both images: the slider doesn't
  describe them.
