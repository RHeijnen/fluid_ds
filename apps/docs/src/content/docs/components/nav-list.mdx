`<fluid-nav-list>` is a vertical navigation menu: a `<nav>` landmark wrapping a
`role="list"` of `<fluid-nav-item>` links. Each item is a real `<a>`, so the
keyboard behaviour is entirely native. Set `current` on the item for the page
the user is on, it applies `aria-current="page"` and the active style.

This is **page navigation**, not a menu of actions. Reach for
[`<fluid-menu>`](/components/menu/) when the items run commands, and
[`<fluid-tree>`](/components/tree/) when the items express a parent/child
hierarchy.

## When to use

- A sidebar that links between the top-level sections of an app.
- An on-page section list (a "what's on this page" rail).
- Any vertical set of links where one item marks the current location.

Reach for [`<fluid-menu>`](/components/menu/) when the items run actions rather
than navigate, and [`<fluid-tree>`](/components/tree/) when the items nest into a
parent/child hierarchy.

## Theming

Every styled property reads a **component-scoped `--fluid-nav-list-*` /
`--fluid-nav-item-*` token that falls back to a main semantic var** (the override
ladder). The status tones are theme-independent; brand themes only retune the
`--fluid-accent-*` track that drives the current-item highlight. The **complete
list** is in the [API table below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-nav-item-fg: var(--fluid-text-secondary);
  --fluid-nav-item-hover-bg: var(--fluid-surface-muted);
  --fluid-nav-item-active-fg: var(--fluid-accent-base);
  --fluid-nav-item-radius: var(--fluid-radius-md);
}
```

### Beyond tokens: `::part()`

The list exposes `part="base"` (the `<nav>`) and `part="list"`; each item exposes
`part="base"`, `part="link"`, `part="icon"`, `part="label"`, and `part="badge"`.
Reach them with [`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part)
for any CSS a token does not cover.

## Accessibility

- The list is a `<nav>` landmark. Give it a unique `label` so it carries an
  accessible name when more than one navigation region exists on the page
  ([SC 1.3.1], [SC 2.4.1]).
- The container is `role="list"` and each item is `role="listitem"`, so the
  count and structure are conveyed to assistive tech.
- Each item is a real `<a href>`, so Tab / Shift+Tab move between links and
  Enter follows the link. No special keys to learn ([SC 2.1.1]).
- The current item carries `aria-current="page"`, conveying location both
  visually (the active style) and programmatically ([SC 1.3.1]).
- The focus ring reads the conformance tokens, so it strengthens automatically
  under `data-fluid-conformance="aaa"`, and the row height floors on
  `--fluid-target-min` so it lifts to a 44×44 target in AAA ([SC 2.5.8]).

### AA vs AAA

By default the component meets WCAG 2.2 **Level AA**: a visible focus ring, a
24×24 minimum target, and text contrast at 4.5:1. Set
`data-fluid-conformance="aaa"` on an ancestor and, with no per-component code,
the focus ring widens (`--fluid-focus-ring-width`) and each link grows to a
44×44 target (`--fluid-target-min`) for **Level AAA** target size (SC 2.5.5) and
focus appearance (SC 2.4.13).
