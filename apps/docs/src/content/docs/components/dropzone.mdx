`<fluid-dropzone>` is a file upload area: a keyboard-focusable region that opens
the native file dialog on click or Enter / Space and also accepts drag-and-drop.
It validates each file against `accept` and `maxSize`, lists the accepted files
with name, size, and a remove button (images get a thumbnail), and reports the
results through **`fluid-change`** and **`fluid-reject`**. It is not
form-associated: it hands you the `File` objects so you own the upload.

</Demo>

### Restrict the type

`accept` mirrors the native attribute: a comma-separated list of MIME types
(`image/png`), wildcards (`image/*`), or extensions (`.pdf`). Files that don't
match fire **`fluid-reject`** with `reason: "type"`. Image files show a
thumbnail.

<Demo>
  <fluid-dropzone accept="image/*" multiple label="Drop images here"></fluid-dropzone>
</Demo>

### Limit the size

`max-size` is the per-file ceiling in bytes. Oversize files fire
**`fluid-reject`** with `reason: "size"`.

## Listening for fluid-change

`<fluid-dropzone>` fires **`fluid-change`** whenever the accepted-file set
changes (a successful drop / pick, or a removal). `event.detail.files` is the
current array of `File` objects. It fires **`fluid-reject`** with
`{ files, reason }` for anything that fails validation.

<Tabs syncKey="framework">
  <TabItem label="HTML">
    ```html
    <fluid-dropzone id="dz" multiple accept="image/*"></fluid-dropzone>

    <script type="module">
      const dz = document.getElementById("dz");
      dz.addEventListener("fluid-change", (e) => {
        console.log(e.detail.files); // [File, File, …]
      });
      dz.addEventListener("fluid-reject", (e) => {
        console.warn(e.detail.reason, e.detail.files); // "type" | "size"
      });
    </script>
    ```
  </TabItem>
  <TabItem label="React">
    ```tsx title="Uploader.tsx"
    import { useRef, useEffect, useState } from "react";
    import "@fluid-ds/components/define/dropzone";

    export function Uploader() {
      const ref = useRef<HTMLElement>(null);
      const [files, setFiles] = useState<File[]>([]);
      useEffect(() => {
        const el = ref.current!;
        const onChange = (e: any) => setFiles(e.detail.files);
        el.addEventListener("fluid-change", onChange);
        return () => el.removeEventListener("fluid-change", onChange);
      }, []);
      return <fluid-dropzone ref={ref} multiple accept="image/*" />;
    }
    ```
  </TabItem>
  <TabItem label="Vue">
    ```vue title="Uploader.vue"
    <script setup lang="ts">
    import { ref } from "vue";
    import "@fluid-ds/components/define/dropzone";
    const files = ref<File[]>([]);
    </script>

    <template>
      <fluid-dropzone
        multiple
        accept="image/*"
        @fluid-change="files = $event.detail.files"
      />
    </template>
    ```
  </TabItem>

## Theming

Every styled property reads a **component-scoped `--fluid-dropzone-*` token that
falls back to a main semantic var** (the override ladder), so you can retheme
one instance, every dropzone, or the whole brand. The **complete list** is in
the [API table below](#api).

```css title="brand.css"
/* Brand-wide: every dropzone */
[data-fluid-brand="custom"] {
  --fluid-dropzone-bg: var(--fluid-surface-base);
  --fluid-dropzone-border: var(--fluid-border-default);
  --fluid-dropzone-accent: var(--fluid-accent-base);
}

/* One instance */
fluid-dropzone[data-fluid-id="avatar"] {
  --fluid-dropzone-radius: var(--fluid-radius-full);
}
```

  Preview and copy token overrides live in the [Theme builder](/playground/).
</Aside>

### Beyond tokens: `::part()`

The shell is [`part="base"`](#api), with `part="dropzone"` for the focusable
region, `part="icon"` and `part="label"` for the prompt, `part="list"` /
`part="file"` for the selected files, `part="thumb"` for the preview, and
`part="remove"` for each file's remove button. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## When to use

- Avatar / cover image upload with an instant thumbnail.
- Attaching one or more documents to a form or message.
- Bulk import (CSV, images) where drag-and-drop is faster than a file picker.

### When *not* to use

- For a single inline file field with no preview, a native
  [`<fluid-input>`](/components/input/) of `type="file"` (or the platform input)
  is lighter.
- To capture free-text tokens / chips, reach for
  [`<fluid-tag-input>`](/components/tag-input/).

## Accessibility

The drop region maps to the WAI-ARIA **button pattern**: a `role="button"` with
`tabindex="0"` that activates on Enter and Space, backed by a visually hidden
native `<input type="file">` that does the actual picking. See the
[accessibility guide](/guides/accessibility/) for the cross-component contract.

### Keyboard

| Key | Action |
| --- | --- |
| `Tab` | Move focus to the drop region, then to each file's remove button. |
| `Enter` / `Space` | Open the native file dialog. |

The drop region and remove buttons show a `:focus-visible` ring that reads the
focus-ring tokens. After a file is removed, focus returns to the drop region so
keyboard users are never stranded.

### Names

The `label` is the drop region's accessible name (`aria-label`) as well as its
visible prompt; slotted content replaces the visible prompt only. Each remove
button is named `Remove <filename>`.

### State semantics

`disabled` sets `aria-disabled="true"`, removes the region from the tab order
(`tabindex="-1"`), and blocks click, keyboard, and drop ingestion. The
dragover highlight is conveyed by both a color and a border-style change, not by
color alone.

### Motion & target size

The border / background transition honors `prefers-reduced-motion: reduce` and
the `--fluid-motion` scalar. The drop region and remove buttons floor on
`--fluid-target-min`, so they meet the 24×24 (AA) target minimum.

### AA vs AAA

The component ships WCAG 2.2 AA by default and scales to AAA via
`data-fluid-conformance="aaa"` (the header toggle is live, and the example
snippets switch with it). It reads the conformance tokens rather than
hard-coding sizes:

| Token | AA | AAA | Criterion |
| --- | --- | --- | --- |
| `--fluid-target-min` | 24px | **44px** | 2.5.8 → 2.5.5 |
| `--fluid-focus-ring-width` | 2px | **3px** | 2.4.7 → 2.4.13 |
