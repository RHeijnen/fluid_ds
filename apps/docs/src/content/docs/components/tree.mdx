`<fluid-tree>` is the hierarchical list view, file explorers, settings trees,
navigation drawers with sub-items. Children are `<fluid-tree-item>` elements,
nested arbitrarily deep. The tree coordinates single selection, manages roving
focus, and implements the WAI-ARIA tree pattern.

## When to use

- File-explorer-style navigation.
- Settings or category hierarchies that benefit from inline expansion.
- For flat top-level navigation, use
  [`<fluid-tabs>`](/components/tabs/) or
  [`<fluid-breadcrumb>`](/components/breadcrumb/).
- For collapsible disclosures that aren't hierarchical, use
  [`<fluid-accordion>`](/components/accordion/).

## Listening for changes

`<fluid-tree>` fires **`fluid-select`** with `{ item }` in `detail` when a node
is activated; each item also fires **`fluid-toggle`** (`{ expanded }`) when it
expands or collapses.

  </TabItem>
  <TabItem label="React">
    ```tsx title="FileTree.tsx"
    import { useRef, useEffect } from "react";
    import "@fluid-ds/components/define/tree";

    export function FileTree() {
      const ref = useRef<HTMLElement>(null);
      useEffect(() => {
        const el = ref.current!;
        const onSelect = (e: Event) =>
          console.log((e as CustomEvent).detail.item);
        el.addEventListener("fluid-select", onSelect);
        return () => el.removeEventListener("fluid-select", onSelect);
      }, []);
      return (
        <fluid-tree ref={ref}>
          <fluid-tree-item>Docs</fluid-tree-item>
        </fluid-tree>
      );
    }
    ```

## Theming

Every styled property reads a **component-scoped token that falls back to a main
semantic var** (the override ladder): the tree exposes `--fluid-tree-*` and each
node exposes `--fluid-tree-item-*` (the selected accent falls through to
`--fluid-accent-base`). The **complete list** is in the [API tables
below](#api).

```css title="brand.css"
[data-fluid-brand="custom"] {
  --fluid-tree-indent: 1.25rem;
  --fluid-tree-item-hover-bg: var(--fluid-surface-muted);
  --fluid-tree-item-selected-accent: var(--fluid-accent-base);
}
```

### Beyond tokens: `::part()`

Each node exposes [`part="base"`](#api) (the row), `part="label"`, and
`part="children"`. Reach them with
[`::part()`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part) for CSS
not covered by a token.

## Accessibility

Implements the WAI-ARIA Tree View pattern. The cross-component contract is in
the [Accessibility guide](/guides/accessibility/).

### Keyboard

| Key | Action |
| --- | --- |
| <kbd>↑</kbd> / <kbd>↓</kbd> | Move between visible items. |
| <kbd>→</kbd> | Expand the focused item (or move to its first child if already expanded). |
| <kbd>←</kbd> | Collapse (or move to the parent if already collapsed). |
| <kbd>Home</kbd> / <kbd>End</kbd> | Jump to the first / last visible item. |
| <kbd>Enter</kbd> | Activate the item (fires `fluid-select`). |

### Names & roles

The host is `role="tree"`; each item is `role="treeitem"` with `aria-expanded`
(parents), `aria-selected`, and `aria-level`
([SC 4.1.2](https://www.w3.org/WAI/WCAG22/Understanding/name-role-value)).

### Motion & target size

- The chevron rotation honors `prefers-reduced-motion`.
- Each row floors its height to `--fluid-target-min`: ≥ **24px** (AA) / **44px**
  (AAA).

### AA vs AAA

Ships **WCAG 2.2 AA**; an ancestor `[data-fluid-conformance="aaa"]` lifts:

| Token | AA | AAA | Criterion |
| --- | --- | --- | --- |
| `--fluid-target-min` | 24px | **44px** | 2.5.8 → [2.5.5 Target Size (Enhanced)](https://www.w3.org/WAI/WCAG22/Understanding/target-size-enhanced) |
| `--fluid-focus-ring-width` | 2px | **3px** | 2.4.7 → [2.4.13 Focus Appearance](https://www.w3.org/WAI/WCAG22/Understanding/focus-appearance) |
